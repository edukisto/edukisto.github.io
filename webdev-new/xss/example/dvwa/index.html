DVWA

Установка​

  • Git
  • Linux
  • macOS
  • Windows 11

Клонируем репозиторий в каталог dvwa.

git clone --depth=1 https://github.com/digininja/DVWA.git dvwa

  1. Скачаем архив, сохраним его как dvwa.zip.

    wget -O dvwa.zip https://github.com/digininja/DVWA/archive/refs/heads/master.zip

  2. Распакуем архив.

    unzip -q dvwa.zip

    Появится каталог DVWA-master.

  3. Переименуем каталог DVWA-master в dvwa.

    mv DVWA-master dvwa

  1. Скачаем архив, сохраним его как dvwa.zip.

    curl -L --output dvwa.zip https://github.com/digininja/DVWA/archive/refs/heads/master.zip

  2. Распакуем архив.

    unzip -q dvwa.zip

    Появится каталог DVWA-master.

  3. Переименуем каталог DVWA-master в dvwa.

    mv DVWA-master dvwa

  1. Скачаем архив, сохраним его как dvwa.zip.

    curl -L --output dvwa.zip https://github.com/digininja/DVWA/archive/refs/heads/master.zip

  2. Распакуем архив.

    tar -xf dvwa.zip

    Появится каталог DVWA-master.

  3. Переименуем каталог DVWA-master в dvwa.

    rename DVWA-master dvwa

Настройка​

  1. Перейдём в каталог dvwa.

    cd dvwa

  2. Создадим конфигурационный файл и приступим к его редактированию.

    • Docker
    • Linux
    • macOS
    • Windows 11

    Ничего не делаем.

    1. Скопируем шаблон конфигурационного файла.

      cp config/config.inc.php.dist config/config.inc.php

    2. Откроем копию на редактирование в VS Code.

      code config/config.inc.php

    1. Скопируем шаблон конфигурационного файла.

      cp config/config.inc.php.dist config/config.inc.php

    2. Откроем копию на редактирование в VS Code.

      code config/config.inc.php

    1. Скопируем шаблон конфигурационного файла.

      copy /v /b config\config.inc.php.dist config\config.inc.php

    2. Откроем копию на редактирование в VS Code.

      code config\config.inc.php

  3. Укажем имя пользователя и пароль.

    $_DVWA[ 'db_user' ]     = getenv('DB_USER') ?: 'dvwa';
    $_DVWA[ 'db_password' ] = getenv('DB_PASSWORD') ?: 'p@ssw0rd';
    $_DVWA[ 'db_user' ]     = getenv('DB_USER') ?: 'pentester';
    $_DVWA[ 'db_password' ] = getenv('DB_PASSWORD') ?: 'ПРИДУМАННЫЙ_РАНЕЕ_ПАРОЛЬ';

  4. Переключим уровень безопасности веб­‑приложения с максимального (делающего взлом невозможным) на минимальный.

    $_DVWA[ 'default_security_level' ] = getenv('DEFAULT_SECURITY_LEVEL') ?: 'impossible';
    $_DVWA[ 'default_security_level' ] = getenv('DEFAULT_SECURITY_LEVEL') ?: 'low';

  5. Отключим веб­‑страницу аутентификации.

    $_DVWA[ 'disable_authentication' ] = getenv('DISABLE_AUTHENTICATION') ?: false;
    $_DVWA[ 'disable_authentication' ] = getenv('DISABLE_AUTHENTICATION') ?: true;

  6. Сохраним изменения в конфигурационном файле.

Запуск​

  • Docker
  • Другие системы
docker compose up --detach
docker compose ps
php -S 127.0.0.1:4280

Использование​

  1. Откройте в браузере веб­‑страницу http://127.0.0.1:4280.

  2. В боковом меню выберите пункт Setup/reset DB. Будет открыта страница Database setup. Нажмите на кнопку Create/reset database, чтобы создать (или воссоздать) БД.

    Database has been created.
    ...
    Setup successful!

  3. В боковом меню выберите пункт DVWA security. Попав на соответствующую веб­‑страницу, выберите из выпадающего списка пункт Low и нажмите на кнопку Submit.

  4. В боковом меню выберите пункт XSS (reflected).
