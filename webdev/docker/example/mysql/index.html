Контейнеризация MySQL

Установка pdo_mysql​

В гостевой ОС необходимо установить пакет с модулем PHP pdo_mysql, представляющим собой драйвер, который обеспечивает взаимодействие PDO с MySQL.

  1. Воспользуйтесь поиском пакетов для ОС Alpine Linux, чтобы найти пакет pdo_mysql для используемой нами версии PHP.

    Скорее всего, это будет пакет php84-pdo_mysql.

  2. Приведите файл php/Dockerfile к следующему виду:

    Файл php/Dockerfile
    FROM php:8.4-fpm-alpine

    RUN cp "${PHP_INI_DIR}/php.ini-development" "${PHP_INI_DIR}/php.ini"

    RUN sed -i 's/max_execution_time = 30/max_execution_time = 180/' "${PHP_INI_DIR}/php.ini"

    RUN apk add --no-cache -- \
            php84-pdo_mysql \
        && docker-php-ext-install \
            pdo_mysql

    Как говорилось ранее, инструкция RUN в Dockerfile выполняет команду в гостевой ОС во время сборки образа. В данном случае посредством менеджера пакетов apk в Alpine Linux будет добавлен (add) пакет php84-pdo_mysql. Этот пакет содержит модуль PHP pdo_mysql. Чтобы установить и активировать его, воспользуемся утилитой docker-php-ext-install, которая встроена в образ php. Кеширование отключено (--no-cache), чтобы apk скачал обновлённый индекс пакетов из Интернета, но не кешировал его и не раздувал тем самым объём образа.

  3. Удалите старый образ.

    docker image remove project:php

  4. Соберите новый образ.

    docker image build --tag=project:php -- ./php

Сборка образа MySQL​

  1. В каталоге проекта создайте каталог mysql.

  2. В каталоге mysql создайте текстовый файл Dockerfile.

  3. Добавьте в Dockerfile инструкцию FROM.

Файл mysql/Dockerfile
FROM mysql:8.4

  1. Подайте команду на сборку нового образа виртуальной машины. Образ будет сохранён на носителе данных для дальнейшего запуска.

    docker image build --tag=project:mysql -- ./mysql/

  2. Подайте команду на запуск нового контейнера.

    • macOS
    • Ubuntu
    • Windows 11
    docker container run \
      --detach \
      --env=MYSQL_ROOT_PASSWORD=1 \
      --name=project-mysql \
      --publish=3306:3306 \
      -- \
      project:mysql
    docker container run \
      --detach \
      --env=MYSQL_ROOT_PASSWORD=1 \
      --name=project-mysql \
      --publish=3306:3306 \
      -- \
      project:mysql
    docker container run ^
      --detach ^
      --env=MYSQL_ROOT_PASSWORD=1 ^
      --name=project-mysql ^
      --publish=3306:3306 ^
      -- ^
      project:mysql

    Опция --env задаёт переменную окружения в гостевой ОС. Такая переменная MYSQL_ROOT_PASSWORD используется СУБД, чтобы назначить пароль суперпользователя.

    Опции --rm на этот раз нет.

    Когда следует отказаться от --rm?

    Создавайте контейнер без опции --rm, если планируете сохранять его состояние между запусками, например, если в файловой системе гостевой ОС размещена БД.

  3. Подайте команду на создание и запуск контейнера.

    • macOS
    • Ubuntu
    • Windows 11
    docker container run \
      --detach \
      --name=project-php \
      --publish=8080:8008 \
      --rm \
      --tty \
      --volume="./src/:/var/www/html/" \
      -- \
      project:php \
      php -S 0.0.0.0:8008
    docker container run \
      --detach \
      --name=project-php \
      --publish=8080:8008 \
      --rm \
      --tty \
      --volume="./src/:/var/www/html/" \
      -- \
      project:php \
      php -S 0.0.0.0:8008
    docker container run ^
      --detach ^
      --name=project-php ^
      --publish=8080:8008 ^
      --rm ^
      --tty ^
      --volume="%CD%\src\:/var/www/html/" ^
      -- ^
      project:php ^
      php -S 0.0.0.0:8008
    Что такое %CD%?

    Переменная %CD% содержит абсолютный путь к текущей рабочей директории. Таким образом, %CD%\src\ это синоним пути C:\Users\ИМЯ_ПОЛЬЗОВАТЕЛЯ_ОС\docker-project\src\.

  4. Попробуйте подключиться из контейнера с PHP к контейнеру MySQL. Для этого разместите в index.php следующий исходный код:

    <?php
    $dsn = 'mysql:host=project-mysql;port=3306;dbname=information_schema';
    $username = 'root';
    $password = '1';

    $connection = new PDO($dsn, $username, $password);
    ЛВС

    Обратите внимание, что имя контейнера можно использовать как доменное имя в локальной вычислительной сети Docker’а.

    Порт не нужно пробрасывать в хостовую систему. Соединение между контейнерами будет установлено через ту же ЛВС.

  5. Обратитесь через браузер по веб­‑адресу http://localhost:8080/index.php.

    В браузер должна быть выведена пустая веб­‑страница без сообщений об ошибках.
