Пример выполнения задания

Ознакомление с примером​

  1. Подготовим исходный код веб­‑сайтов.

    • Windows 7 SP 1, 11
    • Ubuntu

    1. Запустим контрольную панель XAMPP.

    2. Нажмём на кнопку Explorer. Будет открыта папка, в которую установлен XAMPP.

    3. Откроем папку htdocs и создадим внутри неё каталоги blog.bank.example и online.bank.example.

    Скопируем каталоги blog.bank.example и online.bank.example в каталог /var/www/.

    sudo cp -R blog.bank.example /var/www/
    sudo cp -R online.bank.example /var/www/

  2. Создадим виртуальные хосты.

    • Windows 7 SP 1, 11
    • Ubuntu

    Найдём в папке, в которую установлен XAMPP, файл apache\conf\extra\httpd-vhosts.conf и добавим в него следующую конфигурацию.

    Файл apache\conf\extra\httpd-vhosts.conf в папке XAMPP
    <VirtualHost *:80>
        ServerName blog.bank.example
        DocumentRoot "C:\xampp\htdocs\blog.bank.example"
    </VirtualHost>

    <VirtualHost *:80>
        ServerName online.bank.example
        DocumentRoot "C:\xampp\htdocs\online.bank.example"
    </VirtualHost>

    1. Подготовим файлы blog.bank.example.conf и online.bank.example.conf.

      Файл blog.bank.example.conf
      <VirtualHost *:80>
          ServerName blog.bank.example
          DocumentRoot "/var/www/blog.bank.example/"
      </VirtualHost>
      Файл online.bank.example.conf
      <VirtualHost *:80>
          ServerName online.bank.example
          DocumentRoot "/var/www/online.bank.example/"
      </VirtualHost>

    2. Разместим файлы blog.bank.example.conf и online.bank.example.conf в каталоге /etc/apache2/sites-available/.

    sudo cp blog.bank.example.conf /etc/apache2/sites-available/
    sudo cp online.bank.example.conf /etc/apache2/sites-available/

  3. Перезапустим Apache после правки конфигурации.

    • Windows 7 SP 1, 11
    • Ubuntu

    1. Переключимся на контрольную панель XAMPP.

    2. Если напротив метки Apache видна кнопка Stop, то нажмём на неё и дождёмся останова сервера.

    3. Нажмём на кнопку Start, размещённую напротив метки Apache.

    sudo a2ensite blog.bank.example.conf
    sudo a2ensite online.bank.example.conf
    sudo systemctl reload apache2

  4. Откроем на редактирование файл, соотносящий IP­‑адреса с доменными именами.

    • Windows 7 SP 1, 11
    • Ubuntu

    1. Запустим приложение «Командная строка» от имени администратора.

    2. Подадим команду

      Notepad "%SystemRoot%\System32\drivers\etc\hosts"

      Соответствующий шаблону файл (обычно это C:\Windows\System32\drivers\etc\hosts) будет открыт в текстовом редакторе «Блокнот».

    1. Запустим эмулятор терминала.

    2. Подадим команду

      sudo nano /etc/hosts

      Файл /etc/hosts будет открыт в текстовом редакторе nano, запущенном от имени суперпользователя.

  5. Добавим в конец файла:

    127.0.0.1 blog.bank.example
    127.0.0.1 online.bank.example

  6. Сохраним изменения и закроем текстовый редактор.

  7. Подадим команды:

    ping blog.bank.example
    ping online.bank.example

    При обмене пакетами с указанными узлами вычислительной сети должен быть выведен IP­‑адрес 127.0.0.1.

    ping вместо nslookup

    Утилита nslookup в Windows (в отличие от Linux) обращается к DNS, не учитывая записи из файла hosts, поэтому она не подойдёт для наших экспериментов.

  8. Откроем в браузере вкладки с http://blog.bank.example/ и http://online.bank.example/.

Файл online.bank.example/login.php
<?php

declare(strict_types=1);

// use BasicCsrfToken\Manager;
// use EncryptionBasedCsrfToken\AES256GCM;
// use EncryptionBasedCsrfToken\Manager;
use HmacBasedCsrfToken\Manager;

spl_autoload_register();

// Проверяем имя пользователя и пароль.
if ($_POST['username'] !== 'user' || $_POST['password'] !== '123') {
    // Если они неправильные, то даём ответ HTTP со статусом 401 Unauthorized.
    http_response_code(401);
    echo 'Запрос не авторизован. Повторите попытку аутентификации.';
    exit();
}

$lifetime = 60 * 60 * 24;

// Отправляем клиенту сеансовую cookie.
if (false === session_start([
    'cookie_domain' => '', // Передавать только на хост­‑первоисточник.
    'cookie_httponly' => true, // Видна только в запросах HTTP, не в JavaScript.
    'cookie_lifetime' => $lifetime, // Срок действия.
    'cookie_path' => '/', // Передавать на все веб­‑адреса хоста.
    'cookie_samesite' => 'Strict', // Lax|None|Strict
    // 'cookie_secure' => true, // Передавать только по защищённым каналам связи.
    // 'name' => 'PHPSESSID', // Имя cookie; по умолчанию равно PHPSESSID.
    // 'referer_check' => $_SERVER['SERVER_NAME'], // Проверять Referer.
    'use_cookies' => true, // Хранить ID сеанса в cookie.
    'use_only_cookies' => true, // Не хранить ID сеанса в URI.
    'use_strict_mode' => true, // Предотвратить session fixation
    // (если клиент прислал неинициализованный ID сеанса,
    // то сервер направит клиенту новый ID сеанса).
])) {
    // В случае ошибки даём ответ HTTP со статусом 500 Internal Server Error.
    http_response_code(500);
    echo 'Не удалось завершить аутентификацию.';
    exit();
}

// Сохраняем имя пользователя в сеансовых данных.
$_SESSION['username'] = $_POST['username'];

// Наделяем пользователя правом выполнять денежные переводы.
$_SESSION['allow_transfer'] = true;

// Считываем секретный ключ.

// Генерируем CSRF­‑токен.
// $manager = new Manager();
// $_SESSION['csrf_token'] = $manager->make();

$key = base64_decode(trim(file_get_contents('.key.txt')));
// $manager = new Manager(new AES256GCM());
$manager = new Manager();
$expires = time() + $lifetime;
$_SESSION['csrf_token'] = $manager->make($expires, $key);

// Выводим ID сеанса.
$sessionId = session_id();
echo "Вы аутентифицированы! ID сеанса: {$sessionId}.";

var_dump($_SESSION);

Смягчение на основе токенов​

  1. Многим алгоритмам смягчения потребуется секретный ключ на стороне сервера, поэтому создадим файл generate_key.php, содержащий следующий сценарий:

    Файл online.bank.example/generate_key.php
    <?php

    // Имя файла с секретным ключом.
    $filename = '.key.txt';

    try {
        // Если файл уже существует...
        if (file_exists($filename)) {
            throw new Exception("Ключ уже записан в файл $filename.");
        }

        // Создаём секретный ключ ⁠— 256­‑битное значение, выданное КСГПСЧ.
        // Кодируем секретный ключ в base64.
        $key = base64_encode(random_bytes(32));

        // Если не удалось записать ключ в файл...
        if (false === file_put_contents($filename, $key, LOCK_EX)) {
            throw new Exception("Не удалось записать ключ в файл $filename.");
        }

        echo "Ключ записан в файл $filename.";
    } catch (Exception $exception) {
        echo $exception->getMessage();
    }

  2. Вызовем этот сценарий:

    php generate_key.php

    Должен появиться файл .key.txt с секретным ключом.

Простейший токен​

Файл online.bank.example/basiccsrftoken/manager.php
<?php

declare(strict_types=1);

namespace BasicCsrfToken;

class Manager
{
    // Метод генерации CSRF­‑токена.
    function make(): string
    {
        // Посредством КСГПСЧ генерируем 256­‑битное значение.
        // Кодируем его в base64.
        return base64_encode(random_bytes(32));
    }

    // Метод проверки CSRF­‑токена.
    function check(string $token): bool
    {
        // Декодируем сообщение из base64.
        $message = base64_decode($token, true);
        if (false === $message) {
            return false;
        }
        // Проверяем хотя бы длину сообщения.
        return strlen($message) === 32;
    }
}

Пример генерации токена:

$manager = new Manager();
... = $manager->make();

Пример проверки:

$manager = new Manager();
if ($manager->check(...)) {
    // ...
}

Шаблон токена, основанного на шифровании​

Пример реализации шаблона на PHP 7.1 и OpenSSL 1.0.1.

Файл online.bank.example/encryptionbasedcsrftoken/manager.php
<?php

declare(strict_types=1);

namespace EncryptionBasedCsrfToken;

class Manager
{
    private $cryptographer;

    public function __construct($cryptographer)
    {
        $this->cryptographer = $cryptographer;
    }

    // Метод генерации CSRF­‑токена.
    function make(int $expires, string $key): string
    {
        $encryptedMessage = $this->cryptographer->encrypt(
            // Конкатенируем ID сеанса с меткой времени,
            // используя «|» как разделитель.
            session_id() . '|' . $expires,

            // Указываем секретный ключ.
            $key
        );

        // Кодируем кортеж в base64 (английские буквы, цифры, «+», «/», «=»).
        return base64_encode("$encryptedMessage|$expires");
    }

    // Метод проверки CSRF­‑токена.
    function check(string $token, string $key): bool
    {
        // Декодируем нешифрованный кортеж из base64.
        $unencTuple = base64_decode($token);

        // Извлекаем шифрованный кортеж и нешифрованную метку времени.
        list($encTuple, $unenc_expires) = explode('|', $unencTuple);

        // Дешифруем кортеж.
        $decTuple = $this->cryptographer->decrypt($encTuple, $key);

        // Извлекаем из дешифрованного кортежа ID сеанса и метку времени.
        list($dec_session_id, $dec_expires) = explode('|', $decTuple);

        // Если метки времени не совпадают или сеанс окончен,
        var_dump($unenc_expires, $dec_expires, time());
        if (($unenc_expires !== $dec_expires) || (time() > $dec_expires)) {
            // то признаём токен недействительным.
            return false;
        }

        // Сравниваем дешифрованный ID сеанса с текущим.
        return $dec_session_id === session_id();
    }
}
Файл online.bank.example/encryptionbasedcsrftoken/aes256gcm.php
<?php

declare(strict_types=1);

namespace EncryptionBasedCsrfToken;

class AES256GCM
{
    private $aad;
    private $iv;
    private $ivLength;
    private $method;
    private $options;
    private $tag;
    private $tagLength;

    public function __construct(
        int $options = OPENSSL_RAW_DATA,
        string $iv = '',
        string &$tag = '',
        string $aad = ''
    ) {
        $this->aad = $aad;
        $this->method = 'aes-256-gcm';
        $this->ivLength = openssl_cipher_iv_length($this->method);
        $this->iv = openssl_random_pseudo_bytes($this->ivLength);
        $this->options = $options;
        $this->tag = $tag;
        $this->tagLength = 16;
    }

    public function decrypt(string $data, string $key): string
    {
        $data = base64_decode($data);
        $iv = substr($data, 0, $this->ivLength);
        $tag = substr($data, $this->ivLength, $this->tagLength);
        $message = substr($data, $this->ivLength + $this->tagLength);
        return openssl_decrypt(
            $message,
            $this->method,
            $key,
            $this->options,
            $iv,
            $tag,
            $this->aad
        );
    }

    public function encrypt(string $message, string $key): string
    {
        $encryptedMessage = openssl_encrypt(
            $message,
            $this->method,
            $key,
            $this->options,
            $this->iv,
            $this->tag,
            $this->aad,
            $this->tagLength
        );
        return base64_encode($this->iv . $this->tag . $encryptedMessage);
    }
}

Пример генерации токена:

// Срок действия сессии.
$lifetime = 60 * 60 * 24;
// Считываем секретный ключ.
$key = base64_decode(trim(file_get_contents('.key.txt')));
$manager = new Manager(new AES256GCM());
$expires = time() + $lifetime;
// Генерируем CSRF­‑токен.
$manager->make($expires, $key)

Пример проверки:

$key = base64_decode(trim(file_get_contents('.key.txt')));
$manager = new Manager(new AES256GCM());
if ($manager->check($token, $key)) {
    // ...
}

Шаблон токена, основанного на HMAC​

Файл online.bank.example/hmacbasedcsrftoken/manager.php
<?php

declare(strict_types=1);

namespace HmacBasedCsrfToken;

class Manager
{
    // Метод генерации CSRF­‑токена.
    function make(int $expires, string $key): string
    {
        // Вывод перечня хеш­‑функций, подходящих для создания HMAC.
        // print_r(hash_hmac_algos());

        $hmac = hash_hmac(
            // Указываем желаемый алгоритм.
            'sha3-256',

            // Кодируем ID сеанса в base64 (английские буквы, цифры, «+», «/», «=»)
            // и конкатенируем с меткой времени, используя «|» как разделитель.
            base64_encode(session_id()) . '|' . $expires,

            // Указываем секретный ключ.
            $key
        );
        return base64_encode("$hmac|$expires");
    }

    // Метод проверки CSRF­‑токена.
    function check(string $token, string $key): bool
    {
        // Декодируем сообщение из base64.
        $message = base64_decode($token);

        // Извлекаем из сообщения метку времени.
        list(, $expires) = explode('|', $message);

        // Если сеанс окончен,
        if (time() > $expires) {
            // то признаём токен недействительным.
            return false;
        }

        // Сравниваем токен с эталоном.
        return $token === $this->make((int) $expires, $key);
    }
}

Примеры генерации и проверки токена такие же, как в предыдущем случае, но менеджеру не нужен внешний алгоритм шифрования.

$manager = new Manager();
