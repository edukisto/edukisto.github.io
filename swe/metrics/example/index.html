Пример выполнения задания

Подготовка ПО​

  1. Установим модули Python.

    • Windows
    • Ubuntu
    python -m pip install radon
    python3 -m pip install radon

  2. Реализуем функцию вычисления расстояния Хэмминга. Функция определена для двух последовательностей одинаковой длины и равна числу позиций, в которых последовательности различаются. Например, расстояние Хэмминга для чисел 101 и 110 равно 2, поскольку эти числа различаются значениями двух разрядов.

    hamming_distance.py
    """Модуль Хэмминга."""


    def hamming_distance(string1: str, string2: str) -> int:
        """Вычислить расстояние Хэмминга."""
        if len(string1) != len(string2):
            raise ValueError('Строки разной длины')
        # Обнуляем счётчик.
        counter = 0
        for index in reversed(range(len(string1))):
            # Позиции индексированы числами от 0 до (L − 1), где L — длина строки.
            if string1[index] != string2[index]:
                counter += 1
        return counter
Калькулятор для программистов

Для вычислений можно применять веб­‑сервис Wolfram Alpha.

Число строк кода (LOC)​

  1. Подадим команду:

    • Windows
    • Ubuntu
    python -m radon raw hamming_distance.py
    python3 -m radon raw hamming_distance.py
    К сведению

    Если требуется найти суммарное число строк кода для множества файлов, то следует добавить опцию --summary и передать пути к этим файлам:

    • Windows
    • Ubuntu
    python -m radon raw --summary module_a.py module_b.py module_c.py
    python3 -m radon raw --summary module_a.py module_b.py module_c.py

    или

    • Windows
    • Ubuntu
    python -m radon raw --summary *.py
    python3 -m radon raw --summary *.py

  2. Разберём вывод:

    • LOC — общее число строк исходного кода, влючая строки комментариев, документирующие и пустые строки. Вычисляется как сумма SLOC, Single comments, Multi, Blank;
    • LLOC — число предложений независимо от того, сколько строк они занимают; учитывает документирующие блоки (разумеется, тоже независимо от числа занимаемых строк); игнорирует закомментированные и пустые строки;
    • SLOC — число строк исходного кода, исключая строки комментариев, документирующие и пустые строки;
    • Comments — число комментариев, включая внутристрочные (в конце строк); игнорирует документирующие строки;
    • Single comments — число строк комментариев и однострочных документирующих блоков (именно из­‑за учёта последних значение может быть больше Comments);
    • Multi — число строк, включённых только в многострочные документирующие блоки;
    • Blank — число пустых строк;
    • (C % L) — процентное отношение Comments к LOC («Какой процент строк содержит комментарии?»);
    • (C % S) — процентное отношение Comments к SLOC («Каково процентное отношение комментариев к числу строк, влияющих на выполнение программы?»);
    • (C + M % L) — процентное отношение суммы Comments и Multi к SLOC.

    Строки, заполненные исключительно пробелами, также считаются пустыми.

  3. Найдём число PSS, для чего из LOC вычтем Blank. Число PSS равно 14.

  4. Число LSS для всех предложений, кроме комментариев [[^ieee-1045-1992], с. 7], соответствует числу LLOC в Radon и равно 12.

Меры измерения Холстеда​

  1. Изучая исходный код, находим в нём унарные (+xxx, -xxx, ~xxx) и бинарные (%, *, **, +, -, /, //, @) арифметические, логические (!=, <, <=, ==, >, >=, and, in, is, is not, not, not in, or), битовые (&, <<, >>, ^, |) операторы. Скобки, оператор присваивания =, условный оператор (if...else), операторы цикла (for, while) и т. п. не учитываем.

  2. Подсчитываем операторы, учитывая повторы. Оператор != встречается 2 раза (в строках 7 и 14), > — 1 раз (в строке 11), -= — 1 раз (в строке 12), += — 1 раз (в строке 15). Общее число операторов N1=5\Nu_{1} = 5N1​=5.

  3. Подсчитываем операторы, исключая повторы. В результате учитываем четыре оператора: 1) !=; 2) >; 3) -=; 4) +=. Число различных операторов η1=4\eta_{1} = 4η1​=4.

  4. Изучая исходный код, находим в нём операнды (величины, над которыми выполняются операции).

  5. Подсчитываем операнды, учитывая повторы: 1) index встречается 3 раза (в строках 7, 11, 12; в строках 6 и 14 переменная не является операндом); 2) len(string2) — 1 раз; 3) 0 — 1 раз (в строке 11; в строке 10 — не операнд); 4) 1 — 2 раза; 5) string1[index] — 1 раз; 6) string2[index] — 1 раз; 7) counter — 1 раз (в строке 15; в строках 10 и 16 — не операнд). Общее число операндов N2=10\Nu_{2} = 10N2​=10.

  6. Подсчитываем операнды, исключая повторы: 1) index; 2) len(string2); 3) 0; 4) 1; 5) string1[index]; 6) string2[index]; 7) counter. Число различных операндов η2=7\eta_{2} = 7η2​=7.

  7. Вычисляем размер словаря программы:

    η=η1+η2=4+7=11.\eta = \eta_{1} + \eta_{2} = 4 + 7 = 11.η=η1​+η2​=4+7=11.

  8. Вычисляем длину программы:

    N=N1+N2=5+10=15.\Nu = \Nu_{1} + \Nu_{2} = 5 + 10 = 15.N=N1​+N2​=5+10=15.

  9. Вычисляем расчётную длину программы:

    N^=η1log⁡2η1+η2log⁡2η2;\widehat\Nu = \eta_{1} \log_{2}\eta_{1} + \eta_{2} \log_{2}\eta_{2};N=η1​log2​η1​+η2​log2​η2​; N^≈4log⁡24+7log⁡27≈4×2+7×2,8074≈27,65.\widehat\Nu \approx 4 \log_{2}4 + 7 \log_{2}7 \approx 4 \times 2 + 7 \times 2,8074 \approx 27,65.N≈4log2​4+7log2​7≈4×2+7×2,8074≈27,65.

  10. Вычисляем объём программы:

    V=Nlog⁡2η;V = \Nu \log_{2}\eta;V=Nlog2​η; V≈15log⁡211≈15×3,45943≈51,8915.V \approx 15 \log_{2}11 \approx 15 \times 3,45943 \approx 51,8915.V≈15log2​11≈15×3,45943≈51,8915.

    Значение VVV удовлетворяет условию 20⩽V⩽100020 \leqslant V \leqslant 100020⩽V⩽1000 — функция решает не слишком мало и не слишком много задач.

  11. Вычисляем сложность программы:

    D=η12×N2η2;D = \frac{\eta_1}{2} \times \frac{\Nu_2}{\eta_2};D=2η1​​×η2​N2​​; D≈42×107≈2,8571.D \approx \frac{4}{2} \times \frac{10}{7} \approx 2,8571.D≈24​×710​≈2,8571.

  12. Вычисляем усилия:

    E=D×V;E = D \times V;E=D×V; E≈2,8571×51,8915≈148,26;E \approx 2,8571 \times 51,8915 \approx 148,26;E≈2,8571×51,8915≈148,26;

  13. Вычисляем время программирования:

    T=E18;T = \frac{E}{18};T=18E​; T≈148,2618≈8,2367.T \approx \frac{148,26}{18} \approx 8,2367.T≈18148,26​≈8,2367.

  14. Вычисляем число ошибок в программе по формуле Холстеда:

    B=E233000;B = \frac{E^\frac{2}{3}}{3000};B=3000E32​​; B≈148,26233000≈0,0093.B \approx \frac{148,26^\frac{2}{3}}{3000} \approx 0,0093.B≈3000148,2632​​≈0,0093.

  15. Вычисляем число ошибок в программе по упрощённой формуле, которую используют в Radon [1] и программных продуктах IBM:

Цикломатическая сложность​

Подадим команду на вычисление цикломатической сложности.

  • Windows
  • Ubuntu
python -m radon cc --show-complexity hamming_distance.py
python3 -m radon cc --show-complexity hamming_distance.py

Должно быть выведено:

hamming_distance.py
    F 4:0 hamming_distance - A (4)

Цикломатическая сложность v(G′)v(G')v(G′) равна 4.

Мак­‑Кейб указал, что программы с цикломатической сложностью выше 10 проблемны в сопровождении [2, с. 314]. Э. Ван Дорен [3, с. 147] развил идею, составив следующую таблицу:

УсловиеСопровождаемость
1⩽v(G′)⩽101 \leqslant v(G') \leqslant 101⩽v(G′)⩽10Простая программа, без особого риска
11⩽v(G′)⩽2011 \leqslant v(G') \leqslant 2011⩽v(G′)⩽20Более сложная программа, средний риск
21⩽v(G′)⩽5021 \leqslant v(G') \leqslant 5021⩽v(G′)⩽50Сложная программа, высокий риск
v(G′)>50v(G') > 50v(G′)>50Программа непригодна для тестирования, очень высокий риск

Согласно таблице Ван Дорена, наша программа — простая, риск её сопровождения невелик.
