Пример выполнения задания

Установка редактора​

  • Windows 11
  • Ubuntu 22.04.3
  • macOS

Запустите CAEPM73pr3-b1773.exe и следуйте инструкциям.

  1. Установите модуль ядра, обеспечивающий доступ к аппаратной виртуализации на базе Intel VT:

    sudo apt install qemu-kvm

  2. Проверьте результат установки:

    kvm-ok

    Должно быть выведено:

    INFO: /dev/kvm exists
    KVM acceleration can be used

  3. Установите менеджер виртуальных машин.

    sudo apt install virt-manager

  4. Перезагрузите компьютер.

  5. Запустите менеджер виртуальных машин:

    virt-manager

  6. Выберите в главном меню пункт Файл ⏵ Создать виртуальную машину.

  7. Настройте доступ в Интернет.

    Драйвер для Windows XP можно скачать здесь. Целесообразно выбрать файл с расширением имени .iso.

    Если файл с расширением имени .iso недоступен, то его можно создать самостоятельно:

    genisoimage -o drivers.iso ИМЯ_СКАЧАННОГО_ФАЙЛА
    # Если нет `genisoimage`, используйте `mkisofs`.

  8. На вкладке IDE CDROM 1 в поле ввода «Путь к коду» указать путь к файлу с расширением имени .iso.

  9. На вкладке NIC установите:

    Создать на базеВиртуальная сеть 'default': NAT
    Модель устройстваe1000
    Состояние связи☑ активно

  10. Запустите гостевую ОС.

  11. Внутри гостевой ОС запустите чтение компакт­‑диска и установите драйвер.

  12. Внутри гостевой ОС запустите приложение «Командная строка» и проверьте соединение с Интернетом:

    ping ya.ru

    Потерянных пакетов быть не должно.

  13. Упакуйте CAEPM73pr3-b1773.exe:

    genisoimage -o pm.iso CAEPM73pr3-b1773.exe
    # Если нет `genisoimage`, используйте `mkisofs`.

  14. На вкладке IDE CDROM 1 в поле ввода «Путь к коду» указать путь к файлу pm.iso.

  15. Запустите чтение компакт­‑диска и следуйте инструкциям по установке в Windows.

Если не получится установить ERwin Process Modeler в Wine (неактивна кнопка для принятия пользовательского соглашения), воспользуйтесь гипервизором Boot Camp для установки и запуска Windows.

Лицензирование​

  1. В меню «Пуск» выберите пункт CA ⏵ Process Modeler, чтобы запустить приложение CA ERwin Process Modeler r7.3.3.1773 (в прошлом — BPwin).

  2. При запуске будет выведено окно с надписью: «License not found. Do you have a license key?». Нажмите в нём на кнопку «Да».

  3. Будет выведено окно License verification.

    Будьте осторожны

    Программный модуль для верификации лицензии чрезвычайно «глючный». Действуйте строго по инструкции. Отменить ошибочные действия может быть сложно.

    1. В списке Component name выберите пункт ERwin Process Modeler.

    2. В поле Use license key введите лицензионный ключ и нажмите на кнопку Add. Ключ появится в таблице License keys.

    3. Нажмите на кнопку Continue.

Создание модели​

  1. Снимите галочку Display this dialog on startup.

  2. Нажмите на кнопку Cancel.

  3. Нажмите на кнопку File ⏵ New... (⎈ Ctrl + N).

  4. Будет открыто окно

    Убедитесь, что нажаты радиокнопки Create model («Создать модель») и Business process (IDEF0) («Бизнес­‑процесс (IDEF0)»).

    В поле Name («Имя») введите имя модели.

  5. Будет открыто окно Properties for new models.

    В поле Author («Автор») введите свои фамилию, имя и отчество.

Русские надписи​

  1. Выберите в главном меню пункт «Model ⏵ Default fonts ⏵ Context activity...».

    Из выпадающего списка Script («Письменность») выберите пункт «Кириллица», установите галочку Change all occurrences («Изменить все вхождения») и нажмите на кнопку OK.

  2. Выберите в главном меню пункт «Model ⏵ Default fonts ⏵ Frame user text...».

    Из выпадающего списка Script выберите пункт «Кириллица», установите галочку Change all occurrences и нажмите на кнопку OK.

  3. Желательно повторить те же действия для всех пунктов в меню «Model ⏵ Default fonts».

Свойства модели​

  1. Выберите в главном меню пункт «Model ⏵ Model properties...».

Диаграмма А − 0​

  1. Дадим имя блоку. Наведите курсор на блок № 0, вызовите контекстное меню и выберите в нём пункт Name....

    Будет открыто модальное диалоговое окно Activity properties.

    На вкладке Name замените текст «Untitled object 0» на «Приготовить хлеб» и нажмите на кнопку OK.

  2. Создадим входную стрелку. На панели инструментов нажмите на кнопку → (Precedence arrow tool).

    Курсор примет форму перекрестия со стрелкой.

    Подведите курсор к левой границе рабочей области диаграммы. Внутри области появится чёрный прямоугольник.

    Нажмите на него один раз. От места щелчка к курсору потянется линия.

    Подведите курсор к левой границе блока. Внутри блока появится чёрный треугольник.

    Нажмите на треугольник.

  3. Создадим метку для входной стрелки. На панели инструментов нажмите на кнопку → (Pointer tool).

    Курсор примет форму указателя (стрелки).

    Наведите курсор на входную стрелку, вызовите контекстное меню и выберите в нём пункт Name....

    Будет открыто модальное диалоговое окно Arrow properties.

    На вкладке Name введите в поле Arrow name слово «» и нажмите на кнопку OK.

  4. По аналогии создадим другие стрелки: входные (слева), выходные (справа), механизмов (снизу), управляющие (сверху).

Диаграмма А0​

  1. Наведите курсор на зелёную иконку на боковой панели, вызовите контекстное меню и выберите в нём пункт Decompose.

    Будет открыто модальное диалоговое окно Activity box count.

    В нём нажмите на радиокнопку IDEF0 и укажите в поле Number of activities in decomposition (0–8) число 3.

    Будут созданы 3 дочерних диаграммы. Ссылки на них будут размещены на боковой панели.

  2. Выполните диаграмму А0 согласно рисунку.

    Чтобы активировать стрелку (например, чтобы подвести её к блоку), необходимо нажать на её наконечник.

    Стрелки можно ветвить, нажимая на них инструментом → (Precedence arrow tool).
