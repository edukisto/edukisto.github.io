Оркестровка контейнеров

  1. Подайте команду на вывод перечня работающих контейнеров:

    docker container list

  2. Подайте команды на останов работающих контейнеров:

    docker container stop project-php
    docker container stop project-mysql
    docker container stop project-phpmyadmin

  3. Подайте команды на удаление всех остановленных контейнеров:

    docker container prune

  4. Убедитесь, что контейнеров (ни работающих, ни остановленных) не осталось:

    docker container list --all

  5. В каталоге проекта создайте файл compose.yaml.

    Файл compose.yaml
    services:
      php:
        build: ./php/
        command: php -S 0.0.0.0:8080
        container_name: project-php
        depends_on:
          - mysql
        ports:
          - "8080:8080"
        volumes:
          - ./src/:/var/www/html/:rw
        working_dir: /var/www/html/
      mysql:
        container_name: project-mysql
        env_file:
          - ./.env
        # expose:
        #   - 3306
        image: mysql:8.4
      phpmyadmin:
        container_name: project-phpmyadmin
        depends_on:
          - mysql
        environment:
          - PMA_HOST=project-mysql
          - PMA_PORT=3306
          - PMA_SSL=1
        image: phpmyadmin:5-apache
        ports:
          - "8081:80"

    Обратите внимание, что вместо IP или доменного имени хоста в PMA_HOST указано имя контейнера с сервером MySQL.

  6. В каталоге проекта создайте файл .env.

    Файл .env
    MYSQL_ROOT_PASSWORD=1

    Пароль к MySQL можно было задать через атрибут environment, но он вынесен в файл .env, который подключён через атрибут env_file. Файлы, имена которых начинаются с точки, многие ОС воспринимают как скрытые. Также можно заставить Git игнорировать такие файлы.

  7. Подайте команду на сборку и запуск контейнеров:

    docker compose up --build --detach --remove-orphans

  8. Обратитесь через браузер по веб­‑адресам http://localhost:8080/index.php и http://localhost:8081/.

  9. Подайте команду на останов и удаление контейнеров:

    docker compose down --remove-orphans --volumes

  10. Подайте команду на вывод перечня работающих контейнеров:

    docker container list

    Перечень должен быть пуст.
