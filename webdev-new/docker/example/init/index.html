Подготовка

Установка Docker​

Скачайте и установите версию Docker, совместимую с архитектурой процессора и операционной системой Вашего компьютера. Регистрироваться не обязательно.

Хостовая ОС

Docker необходимо установить в хостовой, а не гостевой операционной системе.

Возможные ошибки​

Нет места на носителе данных​

Docker требует приблизительно 4,5 ГБ свободного места на носителе данных.

  • Windows 11 (CMD)
  • Windows 11 (PowerShell)

Пример установки на диск D. Команду следует подать из каталога, в котором размещён дистрибутив. Все каталоги, указанные в путях, будут созданы автоматически.

start /w "" "Docker Desktop Installer.exe" install ^
  --accept-license ^
  --installation-dir="D:\Docker\Docker" ^
  --windows-containers-default-data-root="D:\Docker\containers" ^
  --wsl-default-data-root="D:\Docker\wsl"
No trailing slashes!

Установщик Docker запрещает ставить завершающую обратную косую черту в записи пути.

Пример установки на диск D. Команду следует подать из каталога, в котором размещён дистрибутив. Все каталоги, указанные в путях, будут созданы автоматически.

Start-Process -Wait -FilePath "Docker Desktop Installer.exe" -ArgumentList `
  "install", `
  "--accept-license", `
  "--installation-dir=D:\Docker\Docker", `
  "--windows-containers-default-data-root=D:\Docker\containers", `
  "--wsl-default-data-root=D:\Docker\wsl"
No trailing slashes!

Установщик Docker запрещает ставить завершающую обратную косую черту в записи пути.

Проблемы в Windows​

Unexpected WSL error

  1. Docker неработоспособен в гостевой ОС Windows, запущенной, в частности, в хостовой ОС macOS посредством гипервизора Parallels Desktop. Установите Docker непосредственно в хостовой ОС (macOS).

  2. Откройте приложение «Диспетчер устройств», раздел «Системные устройства», и убедитесь, что включены

  • «Перечислитель виртуальных дисков (Майкрософт)»;
  • «Перечислитель виртуальных сетевых адаптеров NDIS».
WSL update failed

При возникновении ошибки WSL update failed («[Автоматическое] обновление WSL не удалось») запустите приложение «Командная строка», убедитесь, что Ваш компьютер подключён к Интернету, и подайте команду на обновление WSL:

wsl --update

  • Ubuntu
  • Windows 11

Запустите приложение «Терминал» (⎈ Ctrl + ⎇ Alt + T).

Запустите приложение «Командная строка» (не PowerShell!).

Начало проекта​

  1. Назначьте домашний каталог пользователя хостовой ОС текущим рабочим каталогом.

    • macOS
    • Ubuntu
    • Windows 11
    cd ~
    cd ~
    cd %UserProfile%

  2. Создайте каталог для Вашего проекта и переключитесь на него.

    mkdir docker-project && cd docker-project

  3. Откройте текущий рабочий каталог в VS Code.

    code .
