Пример выполнения задания

При построении графов потока управления рекомендуется использовать веб­‑сервис app.diagrams.net. Для представления вершин (узлов) графа подходят элементы Start (круг) и Decision (ромб) со вкладки Flowchart («Блок­‑схема»). Для соединения геометрических фигур линиями необходимо

  • перенести нужные фигуры с палитры в рабочую область;
  • найти, но не выделять фигуру среди размещённых в рабочей области;
  • навести курсор мыши на одну из точек соединения этой фигуры (у Start их восемь, у Decision — четыре; они выделены квадратами голубого цвета). Точка будет обведена зелёным кружком;
  • зажать кнопку мыши. За курсором потянется линия со стрелкой;
  • перевести курсор мыши на одну из точек соединения другой фигуры так, чтобы та точка была обведена зелёным кружком;
  • отпустить кнопку мыши;
  • через контекстное меню преобразовать ломаную линию в прямую или кривую.

Вершины (узлы) графа следует надписать соответствующими номерами строк или фрагментами исходного кода. Для этого необходимо двойным щелчком мыши активировать нужную фигуру и вписать текст.

Для хранения диаграмм рекомендуется использовать «Google Диск», для экспорта — векторный графический формат SVG (File ⏵ Export as ⏵ SVG... или Файл ⏵ Экспортировать как ⏵ SVG...).

Программа с условным оператором​

Задача. Создать контрольные примеры для тестирования базового пути программы.

if a == b:
    print('=')
else:
    print('≠')

Решение.

  1. Выполним диаграмму графа потока управления G2G_2G2​, для наглядности помечая вершины соответствующими фрагментами исходного кода.

    Условный оператор if...else определяет две последовательности выполнения в программе и передаёт управление в зависимости от значения логического выражения a=ba = ba=b. Обозначим это решение предикатной вершиной (ромбом) с пометкой «a == b».

    Предложения print('=') и print('≠') не содержат условных операторов, операторов цикла и т. п. и, следовательно, определяют только по одной последовательности выполнения в программе. Обозначим эти предложения операторными вершинами (кружка́ми) с пометками «'='» и «'≠'».

    Выход из программы обозначим операторной вершиной с пометкой «Выход».

    Теперь отразим передачу управления, соединив вершины дугами (линиями со стрелкой). Условный оператор if...else передаёт управление одному из двух операторов вывода, каждый из которых затем отдаёт управление, и программа завершает работу. Следовательно, создаём четыре дуги: от a == b к '=', от a == b к '≠', от '=' к выходу, от '≠' к выходу. Это соответствует правилу, по которому из предикатной вершины выходят две и только две дуги, а из операторной вершины, если она не конечная, — одна и только одна дуга.

  2. Пронумеруем дуги (EEE), вершины (VVV), грани (FFF).

  3. Подсчитаем дуги (e=∣E∣=4e = |E| = 4e=∣E∣=4), вершины (v=∣V∣=4v = |V| = 4v=∣V∣=4, в т. ч. предикатных: v⋄=1v_\diamond = 1v⋄​=1), грани (f=∣F∣=2f = |F| = 2f=∣F∣=2), пути из v1v_1v1​ в v4v_4v4​ (∣⟨v1,v4⃗⟩∣=2|\langle \vec{v_1, v_4} \rangle| = 2∣⟨v1​,v4​​⟩∣=2).

  4. Вычислим цикломатическую сложность.

    c(G2)={e−v+2gv⋄+1f∣⟨v1,v4⃗⟩∣c(G2)={4−4+2×1=21+1=222c(G2)=2\begin{split} c(G_2) & = \begin{cases} e - v + 2g \\ v_\diamond + 1 \\ f \\ |\langle \vec{v_1, v_4} \rangle| \end{cases} \\ c(G_2) & = \begin{cases} 4 - 4 + 2 \times 1 = 2 \\ 1 + 1 = 2 \\ 2 \\ 2 \end{cases} \\ c(G_2) & = 2 \end{split}c(G2​)c(G2​)c(G2​)​=⎩⎨⎧​e−v+2gv⋄​+1f∣⟨v1​,v4​​⟩∣​=⎩⎨⎧​4−4+2×1=21+1=222​=2​

    Для однократного выполнения всех линейно независимых путей необходимы два контрольных примера.

  5. Спроектируем контрольные примеры.

    155'='
    257'≠'

Программа с двумя условными операторами​

Задача. Создать контрольные примеры для тестирования базового пути функции.

Файл if_else_if_else/src/pos_or_neg.py
def pos_or_neg(value: int) -> int:
    """Отрицательное, положительное или ноль?"""
    if value < 0:
        # Отрицательное число.
        return -1
    if not value:
        # Ноль.
        return 0
    # Положительное число.
    return 1

Решение.

  1. Выполним диаграмму графа потока управления G3G_3G3​, для наглядности помечая вершины соответствующими фрагментами исходного кода.

  2. Пронумеруем дуги (EEE), вершины (VVV), грани (FFF).

  3. Подсчитаем дуги (e=∣E∣=7e = |E| = 7e=∣E∣=7), вершины (v=∣V∣=6v = |V| = 6v=∣V∣=6, в т. ч. предикатных: v⋄=2v_\diamond = 2v⋄​=2), грани (f=∣F∣=3f = |F| = 3f=∣F∣=3), пути из v1v_1v1​ в v6v_6v6​ (∣⟨v1,v6⃗⟩∣=3|\langle \vec{v_1, v_6} \rangle| = 3∣⟨v1​,v6​​⟩∣=3).

  4. Вычислим цикломатическую сложность.

    c(G3)={e−v+2gv⋄+1f∣⟨v1,v6⃗⟩∣c(G3)={7−6+2×1=32+1=333c(G3)=3\begin{split} c(G_3) & = \begin{cases} e - v + 2g \\ v_\diamond + 1 \\ f \\ |\langle \vec{v_1, v_6} \rangle| \end{cases} \\ c(G_3) & = \begin{cases} 7 - 6 + 2 \times 1 = 3 \\ 2 + 1 = 3 \\ 3 \\ 3 \end{cases} \\ c(G_3) & = 3 \end{split}c(G3​)c(G3​)c(G3​)​=⎩⎨⎧​e−v+2gv⋄​+1f∣⟨v1​,v6​​⟩∣​=⎩⎨⎧​7−6+2×1=32+1=333​=3​

    Для однократного выполнения всех линейно независимых путей необходимы три контрольных примера.

Файл while_loop/src/cups_on_the_table.py
def cups_on_the_table(count: int) -> list[str]:
    lines = []
    for i in reversed(range(count)):
        lines.append(f'Чашек на столе: {i + 1}.')
    return lines
