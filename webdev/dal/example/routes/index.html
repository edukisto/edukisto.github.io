4. Маршруты

Теперь следует обеспечить клиентам HTTP, в т. ч. браузерам, возможность вызывать методы контроллеров, подавая запросы через стек протоколов Интернета. Для этого мы должны ассоциировать (сопоставить, связать) каждый метод контроллера с некоторым методом HTTP и универсальным идентификатором ресурса (URI) — проще говоря, веб­‑адресом, который указывают в адресной строке браузера, гиперссылке и т. д.

При создании веб­‑адресов мы будем руководствоваться REST — шаблоном проектирования API. Программные системы, реализующие REST, получают определение RESTful. Современные RESTful­‑системы позволяют клиентам манипулировать ресурсами на сервере, подавая запросы по HTTP. При этом метод протокола определяет операцию над ресурсом, а URI идентифицирует ресурс, над которым выполняется операция.

Веб­‑адреса обычно начинаются с существительного в форме множественного числа.

Метод HTTPURIОперация
POST/примеры/Создание примера
GET/примеры/Вывод множества примеров
GET/примеры/{id}/Вывод примера с идентификатором {id}
PATCH/примеры/{id}/Корректировка примера с идентификатором {id}
PUT/примеры/{id}/Замена примера с идентификатором {id}
DELETE/примеры/{id}/Удаление примера с идентификатором {id}

Разумеется, набор операций операций может отличаться от системы к системе. Например, в Laravel контроллер для API не содержит методов для корректировки или удаления множества ресурсов (т. е. ресурсы можно корректировать или удалять только по одному), а контроллер для UI содержит методы для вывода HTML­‑форм.

Приступим к реализации.

  1. Подадим команду на включение маршрутизации для API.

    php artisan install:api

    Согласимся на предложенные миграции.

    В результате буден создан файл routes/api.php, в котором описывают маршруты API без состояния.

  2. Создадим группу маршрутов с префиксом /api/v1/, где v1 указывает на 1‑ю версию интерфейса. Для этого откроем на редактирование файл routes/api.php и добавим в конец следующие строки:

    Файл routes/api.php
    <?php

    use Illuminate\Http\Request;
    use Illuminate\Support\Facades\Route;

    Route::get('/user', function (Request $request) {
        return $request->user();
    })->middleware('auth:sanctum');

    Route::prefix('v1')->group(function () {
        // ...
    });

    Несложно догадаться, что метод group создаёт группу маршрутов (пока пустую), а метод prefix задаёт всем веб­‑адресам в группе префикс v1. Почему не /api/v1/? Потому что ко всем веб­‑адресам, определённым в файле routes/api.php, префикс /api/ будет добавлен автоматически.

Product​

  1. Спроектируем RESTful API для модели Product:

    Метод HTTPШаблон URIДействие
    POST/api/v1/products/Создать запись о товаре
    GET/api/v1/products/Вывести записи о товарах
    GET/api/v1/products/{product}/Вывести запись о товаре
    PUT/api/v1/products/{product}/Корректировать запись о товаре
    DELETE/api/v1/products/{product}/Удалить запись о товаре

    Шаблонное выражение {product} отмечает место, предназначенное для подстановки идентификатора товара, а именно значения первичного ключа из таблицы БД, slug’а и т. п. Таким образом, шаблону /api/v1/products/{product}/ будут соответствовать веб­‑адреса /api/v1/products/1/, /api/v1/products/2/, /api/v1/products/3/ и т. д.

  2. Отредактируем файл routes/api.php — добавим маршруты в группу.

    Файл routes/api.php
    <?php

    use App\Http\Controllers\ProductController;
    use Illuminate\Http\Request;
    use Illuminate\Support\Facades\Route;

    Route::get('/user', function (Request $request) {
        return $request->user();
    })->middleware('auth:sanctum');

    Route::prefix('v1')->group(function () {
        Route::apiResource('products', ProductController::class);
    });

    Метод Route::apiResource сопоставляет методы index, store, show, update, destroy контроллера ProductController с веб­‑адресами, объединёнными префиксом products, и даёт маршрутам информативные имена: products.index, products.store и т. д.

  3. Подадим команду

    php artisan route:list

    Будет выведен перечень всех маршрутов, среди которых будут маршруты, связанные с контроллером ProductController:

    GET|HEAD  api/v1/products ............. products.index › ProductController@index
    POST      api/v1/products ............. products.store › ProductController@store
    GET|HEAD  api/v1/products/{product} ... products.show › ProductController@show
    PUT|PATCH api/v1/products/{product} ... products.update › ProductController@update
    DELETE    api/v1/products/{product} ... products.destroy › ProductController@destroy

    Для наглядности представим эти данные в табличном виде:

    Метод HTTPШаблон URIМетод ProductControllerИмя маршрута
    GET или HEAD/api/v1/products/indexproducts.index
    POST/api/v1/products/storeproducts.store
    GET или HEAD/api/v1/products/{product}/showproducts.show
    PUT или PATCH/api/v1/products/{product}/updateproducts.update
    DELETE/api/v1/products/{product}/destroyproducts.destroy
    Запись веб­‑адресов

    Каждый веб­‑адрес можно записать в виде строковой константы или вызова функции с именем маршрута в качестве аргумента. Например,

    '/api/v1/products/'

    и

    route('products.index')

    — это два варианта записи одного и того же веб­‑адреса, ведущего к перечню товаров в БД.

    Если маршрут содержит шаблонное выражение (например, {product}), то функции route нужно передать второй параметр — массив, ключи которого соответствуют имени внутри шаблонного выражения. Например, ссылку на веб­‑страницу, посвящённую товару с первичным ключом 123, можно записать как

    '/api/v1/products/123/'

    или

    route('products.show', ['product' => 123])

    Разумеется, вызов функции всегда предпочтителен.

Image и Tag​

  1. Отредактируем файл routes/api.php — добавим маршруты в группу.

    Файл routes/api.php
    <?php

    use App\Http\Controllers\ImageController;
    use App\Http\Controllers\ProductController;
    use App\Http\Controllers\TagController;
    use Illuminate\Http\Request;
    use Illuminate\Support\Facades\Route;

    Route::get('/user', function (Request $request) {
        return $request->user();
    })->middleware('auth:sanctum');

    Route::prefix('v1')->group(function () {
        Route::apiResource('images', ImageController::class);
        Route::apiResource('products', ProductController::class);
        Route::apiResource('tags', TagController::class);
    });

  2. Подадим команду:

    php artisan route:list

    Будет выведен перечень всех маршрутов, среди которых будут маршруты, связанные с контроллером ImageController и TagController.
