Пример выполнения задания

Корень веб­‑сайта​

  • Windows
  • Ubuntu
  1. Установите XAMPP.
  2. Запустите контрольную панель XAMPP.
  3. Нажмите на кнопку Explorer. Будет открыт каталог, в котором размещён XAMPP.
  4. Откройте каталог htdocs и создайте внутри него каталог shop.example (он станет корневым каталогом веб­‑сайта).
  5. Запустите VS Code.
  6. Выберите в главном меню пункт «Файл ⏵ Открыть папку...».
  7. В диалоговом окне выделите каталог shop.example и нажмите на кнопку «Открыть».
  8. Создайте в указанном каталоге файл index.html.

  1. Подайте команды:

    sudo add-apt-repository ppa:ondrej/php
    sudo apt update
    sudo apt install php

    mkdir shop.example && cd $_
    php -S 127.0.0.1:8080

  2. Выберите в главном меню пункт «Файл ⏵ Открыть папку...».

  3. В диалоговом окне выделите каталог shop.example и нажмите на кнопку «Открыть».

  4. Создайте в указанном каталоге файл index.html.

Live preview​

В VS Code предустановлено расширение Live preview. Оно в реальном времени отображает редактируемый документ HTML в браузере, избавляя веб­‑разработчика от необходимости сохранять изменения в файле и обновлять отображение вручную.

  1. Переключите редактор VS Code на вкладку с документом HTML.

  2. Выберите в главном меню пункт «Вид ⏵ Палитра команд...» (F1 или ⎈ Ctrl + ⇧ Shift + P в Windows и Linux, ⇧ Shift​⌘ Command​P в macOS). Появится поле ввода команд.

  3. Введите в это поле команду «> Динамический просмотр: предварительный просмотр (внешний браузер)» или «> Live preview: show preview (external browser)».

    Почему именно внешний браузер?

    Внешний браузер в отличие от встроенного в VS Code позволяет беспрепятственно и в полной мере пользоваться интегрированными инструментами веб­‑разработчика.

Файловая иерархия​

Статический веб­‑сайт не имеет единой точки входа, поэтому веб­‑сервер при обработке запроса вызывает объект файловой системы (т. е. каталог или файл), путь к которому указан в URI, а конкретно — в его компоненте path. Например, URI https://shop.example/products/index.html содержит путь /products/index.html. Запрос по такому URI приведёт к вызову файла index.html из каталога products, который, в свою очередь, размещён в корневом каталоге веб­‑сайта.

Если путь не содержит имени файла, то веб­‑сервер вызовет из указанного каталога так называемый индексный файл, имя которого задано в настройках веб­‑сервера и по умолчанию равно index.html. Таким образом, запрос по URI https://shop.example/products/ приведёт к вызову того же файла, что в предыдущем примере.

Если индексный файл не найден, то веб­‑сервер выведет перечень содержимого каталога.

Воспользуемся этими особенностями и создадим следующую файловую иерархию в корневом каталоге веб­‑сайта.

shop.example
├── about
│   └── index.html
│        { Информация об интернет­‑магазине.
├── carts
│   └── index.html
│        { Перечень товаров в корзине покупателя
├── products
│   ├── hp-laptop-15s
│   │   └── index.html
│   │        { Карточка товара HP Laptop 15s
│   ├── iphone-15-pro-max
│   │   └── index.html
│   │        { Карточка товара iPhone 15 Pro Max
│   ├── playstation-5
│   │   └── index.html
│   │        { Карточка товара PlayStation 5
│   └── index.html
│        { Витрина, перечень товаров в интернет­‑магазине
└── index.html
     { Главная страница интернет­‑магазина

Преимущества такой иерархии:

  • Тим Бернерс­‑Ли, изобретатель URI, считает, что URI не должны содержать имён файлов;
  • единообразие уровней файловой иерархии: путь содержит только имена каталогов (за исключением корневого), но не файлов;
  • расширяемость. Например, при расширении раздела с информацией об интернет­‑магазине можно создать пути /about/contacts/ и /about/legal/, добавив в каталог about каталоги contacts и legal, при этом путь /about/ останется нетронутым.

Валидация​

Порядок действий​

  1. Откройте в браузере веб­‑страницу https://validator.w3.org/nu/.
  2. На панели Show («Показывать») поставьте галочку Outline («Структура документа»).
  3. Из выпадающего списка с заголовком «Check by...» («Проверить посредством...») выберите пункт File upload («Загрузка файла»). Появится поле ввода с диалоговой кнопкой.
  4. Нажмите на указанную кнопку и в диалоговом окне выберите файл, содержащий документ HTML.
  5. Нажмите на кнопку Check («Проверить»).
  6. Если документ прошёл проверку, то будет выведено сообщение Document checking completed. No errors or warnings to show («Проверка документа завершена. Ошибок или предупреждений, которые можно было бы показать, нет»). В противном случае внесите в документ исправления и повторите проверку.
  7. Удостоверьтесь, что в разделе Outline («Структура документа») нет элементов с отметкой [missing] («недостающий»). В противном случае повторно изучите раздел, посвящённый заголовкам, внесите в документ исправления и повторите проверку.
  8. Проведите проверку всех документов HTML в проекте.

Словарь ошибок​

Некоторые предупреждения и сообщения об ошибках:

  • Error: the... element/⁠attribute is obsolete — элемент или атрибут устарел и не соответствует стандарту;

  • Warning: the... attribute... is obsolete — атрибут устарел, но не противоречит стандарту;

  • Use CSS instead — обнаружен предназначенный для внешнего оформления элемент (такой как center или font) или атрибут (такой как bgcolor у элемента body, border у элемента table). Используйте вместо него средства CSS;

  • Error: element EcE_cEc​ not allowed as child of element EpE_pEp​ in this context — родительский элемент EpE_pEp​ не должен содержать дочерних элементов EcE_cEc​. Например, подчинённый перечень нельзя размещать непосредственно внутри старшего перечня:

    <ul>
      <li>A</li>
      <li>B</li>
      <ul>
        <li>B‑1</li>
        <li>B‑2</li>
      </ul>
      <li>C</li>
    </ul>.

    Подчинённый перечень следует разместить внутри одного из элементов старшего перечня:

    <ul>
      <li>A</li>
      <li>B
        <ul>
          <li>B‑1</li>
          <li>B‑2</li>
        </ul>
      </li>
      <li>C</li>
    </ul>.
