Пример выполнения задания

В предыдущей работе мы составили сценарии миграции БД — компоненты для описания данных, т. е. для определения, модификации и удаления таблиц, столбцов, ограничений целостности и т. п.

Одновременно со сценариями миграции БД мы создали несколько классов, которые предназначены для манипулирования данными. Настало время их настроить.

Модели​

Компоненты, которые в Laravel названы моделями, образуют слой доступа к данным (англ. data access layer, DAL). Модели — это классы PHP, которые реализуют архитектурный шаблон active record. Его основные принципы:

  • каждый класс соотнесён с таблицей БД, объекты класса — со строками таблицы, публичные свойства класса/⁠объекта — со столбцами таблицы;
  • методы класса/⁠объекта предназначены для вставки, выборки, корректировки и удаления строк из таблицы БД.

Например, класс app/Models/Product.php соотнесён с таблицей products. Создав в оперативной памяти объект класса Product, заполнив его данными и вызвав метод save(), мы создадим в таблице products новую строку. И наоборот, вызвав из класса Product методы find(), where()->first() или where()->get(), мы получим объект или коллекцию объектов, заполненных данными из выбранных строк таблицы.

Разумеется, модели учитывают отношения, определённые в БД. В рассматриваемом случае в БД определены несколько реляционных отношений:

  • таблица products связана с таблицей images отношением «один ко многим»: к каждому продукту может относиться множество изображений (например, фотоснимков);
  • таблица images связана с таблицей products отношением «один к одному»: каждое изображение может относиться к одному и только одному продукту;
  • таблицы products и tags связаны отношением «многие ко многим». Например, продукт «Мороженое» можно отметить многими тегами («Десерт», «Молочный продукт» и т. п.), а тегом «Молочный продукт» можно отметить многие продукты («Йогурт», «Молоко», «Мороженое» и т. п.).

Реализуем эти отношения на уровне моделей.

Product​

  1. Откроем на редактирование файл app/Models/Product.php.

  2. Добавим метод, связывающий модели Product и Image отношением «один ко многим».

    app/Models/Product.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\HasMany;

    class Product extends Model
    {
        use HasFactory;

        public function images(): HasMany
        {
            return $this->hasMany(Image::class);
        }
    }

    Выражение $this->hasMany(Image::class) можно прочитать так: «Этот экземпляр модели Product может одновременно обладать многими экземплярами модели Image».

    Имя метода images представляет собой записанную строчными буквами форму множественного числа слова Image. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым продуктом могут быть связаны многие изображения.

    Что нам это даст?

    Вызывая метод images, можно манипулировать данными в подчинённой таблице. Например, выбрав из БД запись о продукте, можно извлечь связанные с ней записи об изображениях.

    // Извлекаем из таблицы products строку, в которой столбец id равен 123.
    $product = Product::where('id', 123)->firstOrFail();

    // Извлекаем из таблицы images строки, в которых product_id равен 123.
    $images = $product->images()->get();

    В данном случае Laravel подаст два SQL­‑запроса: SELECT * FROM products WHERE id = 123 и SELECT * FROM images WHERE images.product_id = 123. В приложении БД продукт будет представлена объектом $product класса Product, а изображения — коллекцией (массивом) $images с объектами класса Image.

  3. Добавим метод, связывающий модели Product и Tag отношением «многие ко многим».

    app/Models/Product.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsToMany;
    use Illuminate\Database\Eloquent\Relations\HasMany;

    class Product extends Model
    {
        use HasFactory;

        public function images(): HasMany
        {
            return $this->hasMany(Image::class);
        }

        public function tags(): BelongsToMany
        {
            return $this->belongsToMany(Tag::class);
        }
    }

    Выражение $this->belongsToMany(Tag::class) можно прочитать так: «Этот экземпляр модели Product может одновременно принадлежать многим экземплярами модели Tag».

    Имя метода tags представляет собой записанную строчными буквами форму множественного числа слова Tag. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым продуктом могут быть связаны многие теги.

Image​

  1. Откроем на редактирование файл app/Models/Image.php.

  2. Добавим метод, связывающий модели Image и Product отношением «один к одному».

    app/Models/Image.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsTo;

    class Image extends Model
    {
        use HasFactory;

        public function product(): BelongsTo
        {
            return $this->belongsTo(Product::class);
        }
    }

    Выражение $this->belongsTo(Product::class) (именно belongsTo, а не belongsToMany!) можно прочитать так: «Этот экземпляр модели Image в каждый отдельно взятый момент времени может принадлежать одному и только одному экземпляру модели Product».

    Имя метода product представляет собой записанную строчными буквами форму единственного числа слова Product. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым изображением может быть связан один и только один продукт.

Tag​

  1. Откроем на редактирование файл app/Models/Tag.php.

  2. Добавим метод, связывающий модели Tag и Product отношением «многие ко многим».

    app/Models/Tag.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsToMany;

    class Tag extends Model
    {
        use HasFactory;

        public function products(): BelongsToMany
        {
            return $this->belongsToMany(Product::class);
        }
    }

    Выражение $this->belongsToMany(Product::class) можно прочитать так: «Этот экземпляр модели Tag может одновременно принадлежать многим экземплярам модели Product».

    Имя метода products представляет собой записанную строчными буквами форму множественного числа слова Product. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым тегом могут быть связаны многие продукты.

Фабрики​

Перед тестированием приложения целесообразно заполнить БД, но делать это вручную не рекомендуется: во­‑первых, это трудоёмкая задача; во­‑вторых, требования к данным могут измениться со временем, и тогда процесс заполнения придётся повторить. Разрешить проблему помогут фабрики и сеятели.

⁠Factory (англ. фабрика) — это программный компонент, создающий в оперативной памяти объект заданного класса. Фабрика — один из базовых шаблонов проектирования объектно­‑ориентированных систем, который применяют как изолированно, так и в составе шаблонов более высокого уровня: factory method, abstract factory и т. п. Они упрощают будущий refactoring, отделяя друг от друга фрагменты кода, создающие и использующие объекты.

В Laravel фабрики нужны для производства экземпляров моделей. Каждой модели из каталога app/Models соответствует одна и только одна фабрика из каталога database/factories. Фабрика должна содержать как минимум метод definition(), который возвращает ассоциативный массив. В качестве ключей этого массива приводят имена свойств производимого объекта (они обычно совпадают с именами столбцов в таблице БД), а в качестве значений элементов — задаваемые значения свойств. Значения можно указать в исходном коде, извлечь из файла и даже генерировать псевдослучайным образом — для этого в Laravel встроена библиотека Faker (англ. обманщик). Полный перечень её методов доступен на сайте fakerphp.github.io.

К сведению

Объекты, произведённые фабриками, далеко не всегда помещают в БД. Например, фабрики незаменимы при создании объектов­‑заглушек для тестирования.

Рассмотрим фабрику для модели User, которая была создана автоматически при создании приложения.

database/factories/UserFactory.php
<?php

namespace Database\Factories;

use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Str;

/**
 * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\User>
 */
class UserFactory extends Factory
{
    /**
     * Define the model's default state.
     *
     * @return array<string, mixed>
     */
    public function definition(): array
    {
        return [
            'name' => fake()->name(),
            'email' => fake()->unique()->safeEmail(),
            'email_verified_at' => now(),
            'password' => '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi', // password
            'remember_token' => Str::random(10),
        ];
    }

    /**
     * Indicate that the model's email address should be unverified.
     */
    public function unverified(): static
    {
        return $this->state(fn (array $attributes) => [
            'email_verified_at' => null,
        ]);
    }
}

Как мы видим, метод definition() возвращает массив. Ключи этого массива совпадают с именами столбцов таблицы БД, которые заданы в сценарии миграции (см. файл database/migrations/..._create_users_table.php). Здесь не упомянуты столбцы id (значение первичного ключа), created_at и updated_at (дата и время создания и последнего изменения кортежа), поскольку их значения будут автоматически добавлены при вставке в БД. При необходимости их значения также можно указать в фабрике.

Рассмотрим столбцы в совокупности со вставляемыми данными:

  • name — имя пользователя. fake()->name() подберёт псевдослучайные фамилию, имя и отчество, которые будут иногда повторяться;
  • для столбца email метод safeEmail() вернёт псевдослучайный адрес электронной почты, а метод unique() обеспечит его уникальность;
  • email_verified_at — дата и время подтверждения адреса электронной почты. Благодаря методу now() они будут приравнены к текущим;
  • password — хеш пароля. Задан явно;
  • remember_token — токен для запоминания пароля; он будет равен строке из 10 псевдослучайных символов (Str::random(10)).

Настроим фабрики для остальных моделей.

  1. Откроем на редактирование файл config/app.php.

  2. Заменим строку

    config/app.php
    'faker_locale' => 'en_US',

    на

    config/app.php
    'faker_locale' => 'ru_RU',

    Это позволит некоторым (но не всем) методам Faker’а возвращать значения на русском языке.

  3. Откроем на редактирование файл database/factories/ProductFactory.php.

  4. Приведём файл к следующему виду:

    database/factories/ProductFactory.php
    <?php

    namespace Database\Factories;

    use Illuminate\Database\Eloquent\Factories\Factory;

    /**
     * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\Product>
     */
    class ProductFactory extends Factory
    {
        /**
         * Define the model's default state.
         *
         * @return array<string, mixed>
         */
        public function definition(): array
        {
            return [
                // Метод word() возвращает 1 слово.
                // Метод unique() гарантирует, что слово не будет повторяться.
                'name' => $this->faker->unique()->word(),

                // Число из промежутка [0.01; 999.99]; 2 разряда в дробной части.
                'price' => $this->faker->randomFloat(2, 0.01, 999.99),

                // Текст длиной до 15 знаков.
                'description' => $this->faker->text(15),

                // Число из промежутка [1; 127].
                'quantity' => $this->faker->numberBetween(1, 127),

                // Значение задано явно.
                'available' => true,
            ];
        }
    }

  5. Откроем на редактирование файл database/factories/ImageFactory.php.

  6. Приведём файл к следующему виду:

    database/factories/ImageFactory.php
    <?php

    namespace Database\Factories;

    use Illuminate\Database\Eloquent\Factories\Factory;

    /**
     * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\Image>
     */
    class ImageFactory extends Factory
    {
        /**
         * Define the model's default state.
         *
         * @return array<string, mixed>
         */
        public function definition(): array
        {
            // Создаём имя несуществующего графического файла.
            // для заполнение столбца path в таблице images

            // Метод uuid возвращает универсально уникальный идентификатор,
            // например 4a8d7c0a-31e2-375e-8dce-4fdb21a63172.
            $name = $this->faker->unique()->uuid();

            // Метод randomElement возвращает элемент массива,
            // выбранный псевдослучайным образом. Предложим
            // ему массив расширений имён графических файлов.
            $extension = $this->faker->randomElement([
                'gif',
                'jpeg',
                'png',
                'webp',
            ]);

            // Конструируем имя файла. Должно получиться что-то
            // вроде 4a8d7c0a-31e2-375e-8dce-4fdb21a63172.jpeg.
            $filename = "{$name}.{$extension}";

            return [
                'path' => $filename,
            ];
        }
    }

  7. Откроем на редактирование файл database/factories/TagFactory.php.

  8. Приведём файл к следующему виду:

    database/factories/TagFactory.php
    <?php

    namespace Database\Factories;

    use Illuminate\Database\Eloquent\Factories\Factory;

    /**
     * @extends \Illuminate\Database\Eloquent\Factories\Factory<\App\Models\Tag>
     */
    class TagFactory extends Factory
    {
        /**
         * Define the model's default state.
         *
         * @return array<string, mixed>
         */
        public function definition(): array
        {
            return [
                'label' => $this->faker->unique()->word(),
            ];
        }
    }
Некоторые методы Faker’а, сгруппированные по темам

Бизнес: company() — название организации: ООО «МикроМобайлДизайн», ПАО «ЦементТелеком» и т. п.

Места: address() — почтовый адрес, city() — название населённого пункта, country() — краткое наименование страны, streetAddress() — адрес без указания населённого пункта.

Цвет: colorName() — наименование цвета: красный, зелёный, синий; hexColor() — код цвета: #FF0000, #00FF00, #0000FF.

Разное: boolean() — истина или ложь; regexify('[A-Z][a-z]+') — строка, соответствующая регулярному выражению;.

Вспомогательные:

  • unique() обеспечивает уникальность значений. Способ применения: fake()->unique()->colorName();
  • toLower() приводит к нижнему регистру. Способ применения: fake()->toLower(fake()->colorName());
  • toUpper() приводит к нижнему регистру. Способ применения: fake()->toUpper(fake()->colorName()).

Создав фабрику, можно вызвать её через метод factory модели.

Сеятели​

В Laravel сеятели размещены в каталоге database/seeders.

  1. Откроем на редактирование файл database/seeders/ProductSeeder.php.

  2. Приведём файл к следующему виду:

    database/seeders/ProductSeeder.php
    <?php

    namespace Database\Seeders;

    use App\Models\Image;
    use App\Models\Product;
    use App\Models\Tag;
    use Illuminate\Database\Console\Seeds\WithoutModelEvents;
    use Illuminate\Database\Seeder;

    class ProductSeeder extends Seeder
    {
        /**
         * Run the database seeds.
         */
        public function run(): void
        {
            $tags = Tag::all();
            Product::factory()
                ->count(2)
                ->has(Image::factory()->count(3))
                ->hasAttached($tags->isEmpty() ? Tag::factory() : $tags)
                ->create();
        }
    }

  3. Откроем на редактирование файл database/seeders/ImageSeeder.php.

  4. Приведём файл к следующему виду:

    database/seeders/ImageSeeder.php
    <?php

    namespace Database\Seeders;

    use App\Models\Image;
    use App\Models\Product;
    use Illuminate\Database\Console\Seeds\WithoutModelEvents;
    use Illuminate\Database\Seeder;

    class ImageSeeder extends Seeder
    {
        /**
         * Run the database seeds.
         */
        public function run(): void
        {
            Image::factory()
                ->count(3)
                ->for(Product::factory())
                ->create();
        }
    }

  5. Откроем на редактирование файл database/seeders/TagSeeder.php.

  6. Приведём файл к следующему виду:

    database/seeders/TagSeeder.php
    <?php

    namespace Database\Seeders;

    use App\Models\Product;
    use App\Models\Tag;
    use Illuminate\Database\Console\Seeds\WithoutModelEvents;
    use Illuminate\Database\Seeder;

    class TagSeeder extends Seeder
    {
        /**
         * Run the database seeds.
         */
        public function run(): void
        {
            $products = Product::all();
            Tag::factory()
                ->count(1)
                ->hasAttached($products->isEmpty() ? Product::factory() : $products)
                ->create();
        }
    }

  7. Сеятелей можно запускать по одному:

    php artisan db:seed --class=ProductSeeder

    Если класс не указан, сработает DatabaseSeeder. Его метод run() можно переопределить, чтобы запускать все сеятели одной командой.

  8. Откроем на редактирование файл database/seeders/DatabaseSeeder.php.

  9. Приведём файл к следующему виду:

    database/seeders/DatabaseSeeder.php
    <?php

    namespace Database\Seeders;

    use Illuminate\Database\Console\Seeds\WithoutModelEvents;
    use Illuminate\Database\Seeder;

    class DatabaseSeeder extends Seeder
    {
        use WithoutModelEvents;

        /**
         * Seed the application's database.
         */
        public function run(): void
        {
            $this->call([
                ProductSeeder::class,
                // ImageSeeder::class,
                // TagSeeder::class,
            ]);
        }
    }

  10. Подадим команду:

    php artisan db:seed

  11. Убедимся, что БД заполнена правильно.
