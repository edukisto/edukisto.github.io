1. Модели

Компоненты, которые в Laravel называют моделями, образуют в приложении БД слой доступа к данным (англ. data access layer, DAL). Эти модели реализуют архитектурный шаблон active record. Его основные принципы:

  • класс (в данном случае — класс PHP) соотнесён с таблицей БД, объект (экземпляр класса) — со строкой таблицы, публичное свойство класса/⁠объекта — со столбцом таблицы;
  • класс/⁠объект содержит методы, предназначенные для вставки, выборки, корректировки и удаления строк из таблицы БД.

Например, класс app/Models/Product.php соотнесён с таблицей products. Создав в оперативной памяти объект класса Product, заполнив его данными и вызвав метод save(), мы создадим в таблице products новую строку. Вызвав из класса Product методы find(), where()->first() или where()->get(), мы получим объект или коллекцию объектов, заполненных данными из выбранных строк таблицы.

Разумеется, модели в Laravel учитывают отношения, определённые в БД.

В рассматриваемом случае в БД определены несколько реляционных отношений:

  • таблица products связана с таблицей images отношением «один ко многим»: к каждому продукту может относиться множество изображений (например, фотоснимков);
  • таблица images связана с таблицей products отношением «один к одному»: каждое изображение может относиться к одному и только одному продукту (так решил заказчик);
  • таблицы products и tags связаны отношением «многие ко многим». Например, продукт «Мороженое» можно отметить многими тегами («Десерт», «Молочный продукт» и т. п.), а тегом «Молочный продукт» можно отметить многие продукты («Йогурт», «Молоко», «Мороженое» и т. п.).

Реализуем эти отношения на уровне моделей.

Product​

  1. Откроем на редактирование файл app/Models/Product.php.

  2. Добавим метод, связывающий модели Product и Image отношением «один ко многим».

    app/Models/Product.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\HasMany;

    class Product extends Model
    {
        /** @use HasFactory<\Database\Factories\ProductFactory> */
        use HasFactory;

        public function images(): HasMany
        {
            return $this->hasMany(Image::class);
        }
    }

    Выражение $this->hasMany(Image::class) можно прочитать так: «Этот экземпляр модели Product может одновременно обладать многими экземплярами модели Image».

    Имя метода images представляет собой записанную строчными буквами форму множественного числа слова Image. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым продуктом могут быть связаны многие изображения.

    Что нам это даёт?

    Вызывая метод images, можно манипулировать данными в подчинённой таблице. Например, выбрав из БД запись о продукте, можно извлечь связанные с ней записи об изображениях.

    // Извлекаем из таблицы products строку, в которой столбец id равен 123.
    $product = Product::where('id', 123)->firstOrFail();

    // Извлекаем из таблицы images строки, в которых product_id равен 123.
    $images = $product->images()->get();

    В данном случае Laravel подаст два SQL­‑запроса: SELECT * FROM products WHERE id = 123 и SELECT * FROM images WHERE images.product_id = 123. В приложении БД продукт будет представлена объектом $product класса Product, а изображения — коллекцией (массивом) $images с объектами класса Image.

  3. Добавим метод, связывающий модели Product и Tag отношением «многие ко многим».

    app/Models/Product.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsToMany;
    use Illuminate\Database\Eloquent\Relations\HasMany;

    class Product extends Model
    {
        /** @use HasFactory<\Database\Factories\ProductFactory> */
        use HasFactory;

        public function images(): HasMany
        {
            return $this->hasMany(Image::class);
        }

        public function tags(): BelongsToMany
        {
            return $this->belongsToMany(Tag::class);
        }
    }

    Выражение $this->belongsToMany(Tag::class) можно прочитать так: «Этот экземпляр модели Product может одновременно принадлежать многим экземплярами модели Tag».

    Имя метода tags представляет собой записанную строчными буквами форму множественного числа слова Tag. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым продуктом могут быть связаны многие теги.

Image​

  1. Откроем на редактирование файл app/Models/Image.php.

  2. Добавим метод, связывающий модели Image и Product отношением «один к одному».

    app/Models/Image.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsTo;

    class Image extends Model
    {
        /** @use HasFactory<\Database\Factories\ImageFactory> */
        use HasFactory;

        public function product(): BelongsTo
        {
            return $this->belongsTo(Product::class);
        }
    }

    Выражение $this->belongsTo(Product::class) (именно belongsTo, а не belongsToMany!) можно прочитать так: «Этот экземпляр модели Image в каждый отдельно взятый момент времени может принадлежать одному и только одному экземпляру модели Product».

    Имя метода product представляет собой записанную строчными буквами форму единственного числа слова Product. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым изображением может быть связан один и только один продукт.

Tag​

  1. Откроем на редактирование файл app/Models/Tag.php.

  2. Добавим метод, связывающий модели Tag и Product отношением «многие ко многим».

    app/Models/Tag.php
    <?php

    namespace App\Models;

    use Illuminate\Database\Eloquent\Factories\HasFactory;
    use Illuminate\Database\Eloquent\Model;
    use Illuminate\Database\Eloquent\Relations\BelongsToMany;

    class Tag extends Model
    {
        /** @use HasFactory<\Database\Factories\TagFactory> */
        use HasFactory;

        public function products(): BelongsToMany
        {
            return $this->belongsToMany(Product::class);
        }
    }

    Выражение $this->belongsToMany(Product::class) можно прочитать так: «Этот экземпляр модели Tag может одновременно принадлежать многим экземплярам модели Product».

    Имя метода products представляет собой записанную строчными буквами форму множественного числа слова Product. Имя обязано отражать особенности установленного отношения и в данном случае подчёркивает, что с каждым тегом могут быть связаны многие продукты.
