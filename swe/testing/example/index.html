Проектирование тестов​

Основы теории множеств

В решении задач используются математические обозначения для описания множеств.

Обозначения некоторых бесконечных числовых множеств:

  • N={1;2;3;...}\N = \{1; 2; 3; ...\}N={1;2;3;...} — множество натуральных (целых положительных) чисел;
  • N0={0;1;2;3;...}\N_0 = \{0; 1; 2; 3; ...\}N0​={0;1;2;3;...} — множество целых неотрицательных чисел;
  • Z={...;−2;−1;0;1;2;...}\Z = \{...; -2; -1; 0; 1; 2; ...\}Z={...;−2;−1;0;1;2;...} — множество целых чисел;
  • R\RR — множество действительных чисел.

Если элемент xxx принадлежит множеству XXX, то записывают: x∈Xx \in Xx∈X. Например, 2∈{1;2;3}2 \in \{1; 2; 3\}2∈{1;2;3}, а 0,5∈R0,5 \in \R0,5∈R. Если элемент xxx не принадлежит множеству XXX, то записывают: x∉Xx \notin Xx∈/X. Например, −1∉N-1 \notin \N−1∈/N.

Если каждый элемент множества AAA также принадлежит множеству BBB, то AAA называют подмножеством BBB. Например, если A={2;3}A = \{2; 3\}A={2;3}, B={1;2;3}B = \{1; 2; 3\}B={1;2;3}, то A⊂BA \subset BA⊂B. Бесконечные числовые множества также могут являться подмножествами: N⊂R\N \subset \RN⊂R.

Запись вида {x∣f}\{x | f\}{x∣f} следует читать как «множество элементов xxx таких, что выполняется fff». Например, {n∣(n⩾0)∧(n⩽255)}\{n | (n \geqslant 0) \land (n \leqslant 255)\}{n∣(n⩾0)∧(n⩽255)} обозначает множество элементов nnn таких, что выполняется выражение ((n⩾0)∧(n⩽255))((n \geqslant 0) \land (n \leqslant 255))((n⩾0)∧(n⩽255)). Иначе говоря, это множество значений, лежащих в замкнутом интервале [0..255][0..255][0..255]

Запись вида {x∈X∣f}\{x \in X | f\}{x∈X∣f} — это сокращённая запись {x∣(x∈X)∧f}\{x | (x \in X) \land f\}{x∣(x∈X)∧f}. Например, {z∈Z∣z<0}\{z \in \Z | z < 0\}{z∈Z∣z<0} — это множество целых отрицательных чисел ({...;−2;−1}\{...; -2; -1\}{...;−2;−1}).

Задача 1. Дана спецификация программы: программа вычисляет факториал; факториал определён на множестве целых неотрицательных чисел N0\N_0N0​; аргумент nnn вводит пользователь. Спроектировать тестирование программы.

Решение. Определение факториала позволяет идентифицировать три класса эквивалентности:

  1. допустимый V1={n∣n∈N0}=N0V_1 = \{n | n \in \N_0\} = \N_0V1​={n∣n∈N0​}=N0​;

  2. два недопустимых:

    • V2={n∣n<0}V_2 = \{n | n < 0\}V2​={n∣n<0} — множество отрицательных чисел;
    • V3={n∣(n>0)∧(n∉N)}V_3 = \{n | (n > 0) \land (n \notin \N)\}V3​={n∣(n>0)∧(n∈/N)} — множество нецелых положительных чисел.

Примечание 1. В спецификации программы не указано максимальное допустимое значение аргумента nnn. Идентифицировать недопустимый класс эквивалентности {n∣n>∞}\{n | n > \infin\}{n∣n>∞} невозможно, т. к. не существует число nnn такое, что n>∞n > \infinn>∞.

Примечание 2. Действительные числа (R\RR), в т. ч. целые (Z\ZZ) и представленные в виде десятичных дробей, во многих языках программирования могут принадлежать одному типу данных. Примером такого типа служит double, который реализован в C, JavaScript (под именем Number) и т. п. Это не противоречит законам математики, поскольку Z⊂R\Z \subset \RZ⊂R. Однако в данную спецификацию программы не включены какие­‑либо требования к используемому языку программирования или программному интерфейсу. Как следствие, вместо отдельных недопустимых классов эквивалентности {n∣(n<0)∧(n∈Z)}\{n | (n < 0) \land (n \in \Z)\}{n∣(n<0)∧(n∈Z)} (множество целых отрицательных чисел) и {n∣(n<0)∧(n∉Z)}\{n | (n < 0) \land (n \notin \Z)\}{n∣(n<0)∧(n∈/Z)} (множество нецелых отрицательных чисел) идентифицирован обобщённый класс V2={n∣(n<0)}V_2 = \{n | (n < 0)\}V2​={n∣(n<0)}.

Случайным образом выберем по одному элементу класса эквивалентности и разработаем контрольные примеры:

Задача 2. В спецификацию программы вычисления факториала внесены изменения: на аргумент nnn наложены ограничения типа данных unsigned short int по стандарту ISO/⁠IEC 9899:2017. Заново спроектировать тестирование программы, используя методику разбиения [на классы] эквивалентности.

Решение. Если спецификация программы содержит требования к используемому языку программирования или программному интерфейсу (стандарт ISO/⁠IEC/⁠IEEE 29148:2018 это допускает), то тестер может идентифицировать классы эквивалентности, отражающие особенности реализации типов данных.

Разобьём входные данные на классы эквивалентности. Согласно стандарту ISO/⁠IEC 9899:2017 на язык программирования C, значение переменной целочисленного типа unsigned short int лежит в интервале целых чисел [0;216−1][0; 2^{16} - 1][0;216−1]. Таким образом, установлено максимальное допустимое значение аргумента nnn, что заставляет нас заново определить единственный допустимый класс эквивалентности

V1={n∈N0∣n⩽216−1}.V_1 = \{n \in \N_0 | n \leqslant 2^{16} - 1\}.V1​={n∈N0​∣n⩽216−1}.

Следовательно, натуральные числа больше 216−12^{16} - 1216−1 теперь принадлежат недопустимому классу эквивалентности

V4={n∈N0∣n>216−1}.V_4 = \{n \in \N_0 | n > 2^{16} - 1\}.V4​={n∈N0​∣n>216−1}.

Недопустимые классы эквивалентности V2V_{2}V2​ и V3V_{3}V3​ не изменяются.

Задача 3. В спецификацию программы вычисления факториала внесены изменения: n!n!n! должно лежать в интервале, который установлен стандартом ISO/⁠IEC 9899:2017 для целочисленных значений, принадлежащих типу double или unsigned long long int. Заново спроектировать тестирование программы, используя методику разбиения [на классы] эквивалентности.

Решение. Факториал — быстрорастущая функция. Два наиболее ёмких стандартных типа C — double и unsigned long long int — позволяют записать 20!20!20!, но уже 21!21!21! превышает их максимальное допустимое целочисленное значение в 3 раза, а 22!22!22! — в 60 раз.

Таким образом, требования к выходным данным заставляет нас заново определить единственный допустимый класс эквивалентности как множество целых неотрицательных чисел от 0 до 20 включительно:

V1={n∈N0∣n⩽20}.V_{1} = \{n \in \N_{0} | n \leqslant 20\}.V1​={n∈N0​∣n⩽20}.

Следовательно, натуральные (целые положительные) числа больше 202020 теперь принадлежат недопустимому классу эквивалентности

V4={n∈N0∣n>20}.V_{4} = \{n \in \N_{0} | n > 20\}.V4​={n∈N0​∣n>20}.

Недопустимые классы эквивалентности V2V_{2}V2​ и V3V_{3}V3​ не изменяются.

Тестовое покрытие​

  1. Установим coverage.py.

    • Windows
    • Ubuntu
    python -m pip install coverage
    python3 -m pip install coverage

    В окно эмулятора терминала будет выведено

    Collecting coverage
    Downloading coverage...
    Installing collected packages: coverage
    Successfully installed coverage...

    или

    Requirement already satisfied: coverage...

  2. Выведем номер установленной версии coverage.py.

    • Windows
    • Ubuntu
    python -m coverage --version
    python3 -m coverage --version

    В окно эмулятора терминала будет выведено:

    Coverage.py, version... with C extension
    Full documentation is at https://coverage.readthedocs.io

  3. Выполним тесты, измеряя тестовое покрытие.

    • Windows
    • Ubuntu
    python -m coverage run --branch -m unittest discover ./tests/
    python3 -m coverage run --branch -m unittest discover ./tests/

    Опция coverage run -m позволяет выполнить тот или иной модуль Python так же, как это делает -m. Проще говоря, мы запустили модуль coverage, который запустил другой модуль — unittest. Последний по­‑прежнему выполняет тесты, а первый измеряет тестовое покрытие.

    Опция --branch заставляет измерять покрытие не только предложений, но и веток исходного кода.

  4. Выведем отчёт в окно эмулятора терминала.

    • Windows
    • Ubuntu
    python -m coverage report
    python3 -m coverage report

    Будет создан файл базы данных SQLite 3 с именем .coverage.

  5. Сформируем отчёт в HTML.

    • Windows
    • Ubuntu
    python -m coverage html
    python3 -m coverage html

    Будет создан каталог htmlcov.

  6. Откроем в браузере файл htmlcov/index.html.
