Установка Npcap (только для Windows)​

Wireshark без Npcap или устаревшего WinPcap не сможет перехватывать пакеты из возвратной петли (loopback). Скачайте Npcap (не путать с nmap!) с веб­‑страницы https://nmap.org/npcap/#download. При установке должны быть отмечены только следующие check box’ы:

  • Support loopback traffic (“Npcap loopback adapter” will be created) ⁠— «Поддержка трафика возратной петли (будет создан адаптер Npcap для возвратной петли)»;
  • Restrict Npcap driver’s access to administrators only («Разрешить доступ к драйверу Npcap только администраторам»);
  • Support raw 802.11 traffic (and monitor mode) for wireless adapters («Поддержка необработанного трафика 802.11 (и режима мониторинга) для беспроводных адаптеров»);
  • Install Npcap in WinPcap API­‑compatible mode («Установить Npcap в режиме совместимости с прикладным программным интерфейсом WinPcap»).

Установка Wireshark​

Скачайте и запустите дистрибутив Wireshark с веб­‑страницы https://www.wireshark.org/#download. Установщик должен обнаружить Npcap. Установка USBPcap — по желанию. По окончании установки перезагрузите операционную систему.

Сбор информации о ресурсе, полученном по схеме http​

  1. Запустите Wireshark от имени администратора (суперпользователя).
  2. Нажмите на кнопку ⚙ Capture options ⁠— «Опции захвата» в главном меню (Ctrl ⁠+ ⁠K).
  3. В модальном окне Capture interfaces («Интерфейсы захвата») на вкладке Input («Ввод») выделите Npcap Loopback Adapter.
  4. Начните перехват пакетов, нажав Start ⁠— «Старт».
  5. Посредством браузера осуществите переход на любой веб­‑сайт, не применяющий шифрование, например http://localhost/.
  6. Завершите перехват пакетов, нажав Stop capturing packets ⁠— «Остановить захват пакетов» (Ctrl + E).
  7. Установите галочку «Просмотр ⁠→ Панель инструментов „Фильтр“» (View ⁠→ ⁠Filter Toolbar), если она снята.
  8. В поле ввода на панели инструментов «Фильтр» укажите «http» (без кавычек) и нажмите «➞» (Apply the filter string to the display, «Применить эту строку фильтра к отображённому на экране» или ⏎ Enter).
  9. Последовательно нажмите на строки, сожержащие информацию о запросе и ответе, и скопируйте в отчёт перехваченные сообщения HTTP (Копировать ⁠→ Все видимые пункты выбранного дерева).

Примерный результат:

GET / HTTP/1.1
Host: example.com
HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Date: Tue, 19 Jan 2038 03:14:07 GMT

Сбор информации о ресурсе, полученном по схеме https​

Повторите действия из предыдущего задания, но изучите веб­‑сайт, применяющий шифрование, например https://localhost/ и, следовательно, отфильтровывайте пакеты по тегу «tls» (без кавычек). Учитывайте содержимое графы Info, IP­‑адреса отправителя и получателя запроса.

Используя приведённую ниже информацию, внесите в отчёт:

  • версию протокола, предложенную клиентом серверу;
  • перечень комплектов шифров, предложенных клиентом серверу;
  • перечень методов сжатия, предложенных клиентом серверу;
  • информацию о поддержке SNI (определяется по наличию доменного имени);
  • версию протокола, комплект шифров и метод сжатия, выбранные сервером из предложенных клиентом.

Клиент ⁠— серверу: ClientHello

  • client_version (Version в Wireshark) ⁠— версия протокола (которой надлежит быть наивысшей), предпочтительная для клиента. Протокол SSL устарел и опасен. В 2020 г. корпорации Apple, Google, Mozilla, Microsoft также прекратят поддержку в браузерах протокола TLS версий 1.0 и 1.1 (останутся только версии 1.2 и 1.3);
  • random содержит gmt_unix_time (метка времени по UTC) и random_bytes (28 байтов из защищённого ГПСЧ клиента);
  • session_id ⁠— идентификатор сеанса, который клиент желает возобновить при соединении;
  • cipher_suites ⁠— перечень известных клиенту комплектов шифров в порядке снижения приоритета. Наиболее распространены TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA и TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384;
  • compression_methods ⁠— перечень методов сжатия. Должен быть предложен как минимум метод null;
  • extensions ⁠— расширения, в т. ч., возможно, server_name ⁠— расширение Server Name Indication (SNI).

Сервер ⁠— клиенту: ServerHello

  • random, сгенерированный независимо от клиентского;
  • версия протокола (server_version), комплект шифров (cipher_suite) и метод сжатия (compression_method), выбранные сервером из предложенных клиентом;
  • session_id ⁠— идентификатор сессии.

Далее может идти обмен сертификатами.

Сервер ⁠— клиенту: ChangeCipherSpec, Finished, Application Data

Посредством ChangeCipherSpec клиент уведомляет сервер, что последующие записи будут зашифрованы. Finished (отображается как application data) ⁠— первое зашифрованное сообщение, и сервер должен его расшифровать. Последующие записи (application data) также будут зашифрованы.

Клиент ⁠— серверу: ChangeCipherSpec, Finished, Application Data

Подбор шифров​

Часть шифров в OpenSSL сохранена для сохранения обратной совместимости и небезопасна. Попробуем для начала вывести полный перечень шифров и подобрать наиболее актуальные:

openssl ciphers -v 'ALL:COMPLEMENTOFALL' | column -t

Шифры с меткой Au=None (обычно это шифры ADH и AECDH) не предлагают аутентификацию, то есть не проверяют целостность сообщений и создают уязвимость для атак «человек посередине». Шифры с меткой Enc=None (обычно это шифры со словом NULL в идентификаторе) на самом деле не шифруют. Шифры с отметками SSLv3 и TLSv1.

openssl ciphers -v 'TLSv1.2:!aNULL:!eNULL' | column -t

TLSv1.2 обозначает множество шифров, совместимых только с актуальными версиями протокола не ниже TLS 1.2. !aNULL исключает шифры без проверки целостности сообщений. !eNULL исключает шифры без… шифрования.

Идентфикаторы !EXPORT и !LOW, исключающие низкокачественные шифры, допущенные к экспорту из США в 1990‑е годы, и 56‑ и 64­‑битные шифры соответственно, неактуальны для OpenSSL 1.1.0 и выше, но могут использоваться для сохранения обратной совместимости конфигурации.

ssl_ciphers 'TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256:TLS_AES_128_GCM_SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-CHACHA20-POLY1305:ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-SHA384:ECDHE-RSA-AES128-SHA256';

ssl_prefer_server_ciphers on;
ssl_protocols TLSv1.2 TLSv1.3;
