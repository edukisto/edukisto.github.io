Установка и настройка ПО

Установка ПО​

  1. Убедимся, что установлены версии программной среды Node.js и пакетного менеджера npm.

    # Рекомендуется 20.17 или 22.9.
    node -v
    # Рекомендуется 10.8.
    npm -v

    Если указанные программные средства отсутствуют или устарели, обратитесь к инструкции по установке.

  2. Создадим каталог проекта и перейдём в него.

    mkdir react-shop-ui.example
    cd react-shop-ui.example

  3. Инициализируем программный пакет Node.js.

    npm init

    Заполним поля согласно таблице 1, подтверждая ввод нажатием на клавишу ↵ ⁠Enter.

    Данные для заполнения полей
    Описание поляЗначение
    Package namereact-shop-ui
    Version0.1.0
    DescriptionПользовательский веб­‑интерфейс для интернет­‑магазина
    Entry pointindex.jsx
    Test commandjest
    Git repositoryURI, оканчивающийся на .git
    Keywordsreact, shop, ui
    AuthorИмя, фамилия (например, Sergei Karavaev)
    LicenseИдентификатор
    Выбор лицензии

    Для веб­‑приложения с открытым и свободным исходным кодом желательно выбрать лицензию, признанную Фондом свободного ПО (FSF) и Инициативой открытого ПО (OSI; не путать с ISO — Международной организацией по стандартизации и её проектом в области сетевых технологий — OSI). Из числа этих лицензий наиболее популярны MIT (идентификатор MIT) и Apache версии 2.0 (идентификатор Apache-2.0). Обе разрешают копирование, использование, изменение, слияние, публикацию, распространение, продажу и даже сублицензирование производного произведения, что отличает их от GNU GPL и GNU AGPL. Таким образом, мы вправе включить React, распространяемый по лицензии MIT, в коммерческое веб­‑приложение с закрытым исходным кодом.

    Будет выведено сообщение, предлагающее подтвердить выбор.

    Is this OK? (yes)

    Нажмём на клавишу ↵ ⁠Enter.

    В каталоге появится файл package.json.

  4. Установим пакеты, от которых зависит наше веб­‑приложение.

    npm install --save-prod react react-dom
    npm install --save-dev @types/react @types/react-dom vite

    Установленные пакеты будут размещены в локальном каталоге node_modules и будут зарегистрированы в файле package.json, в разделах dependencies (зависимости времени выполнения) и devDependencies (зависимости времени разработки). В файле package-lock.json будут зарегистрированы транзитивные зависимости веб­‑приложения, т. е. зависимости его зависимостей.

  5. Дерево элементов каталога должно выглядеть следующим образом:

    .
    ├── node_modules
    │   └── ...
    ├── package-lock.json
    └── package.json

Поддержка модулей ECMAScript​

JavaScript — это надмножество языка программирования ECMAScript, определяемого стандартом ECMA‑262. В ранних изданиях документа (1–⁠5, 5.1) модули не были предусмотрены, что привело к появлению в JavaScript нестандартных систем экспорта и импорта модулей: CommonJS в Node.js, библиотека RequireJS для браузеров и т. п.

Нативные модули были определены в 2015 г. в 6‑м издании ECMA‑262, но их использование до сих пор сопряжено с рядом трудностей из­‑за необходимости сохранить обратную совместимость.

  1. Добавим в файл package.json нештатное поле type.

    "type": "module"

  2. Создадим файл tsconfig.json с конфигурацией проекта, использующего возможности TypeScript — строго типизированного надмножества JavaScript. Конфигурация позволяет проверять синтаксис включённых в проект файлов JavaScript и JSX (allowJs, checkJs) и обеспечивает автоматизированный импорт модулей, совместимый с ECMAScript 2022 и выше ("module": "ESNext").

    Вариант с базовым набором опций:

    {
      "compilerOptions": {
        "allowJs": true,
        "checkJs": true,
        "jsx": "react-jsx",
        "module": "ESNext",
        "moduleResolution": "Bundler"
      },
      "include": [
        "**/*.jsx"
      ]
    }

    Вариант с расширенным набором опций (максимально строгие проверки синтаксиса):

    tsconfig.json
    {
      "compilerOptions": {
        "allowJs": true,
        "allowUnreachableCode": false,
        "allowUnusedLabels": false,
        "checkJs": true,
        "composite": false,
        "downlevelIteration": true,
        "esModuleInterop": true,
        "exactOptionalPropertyTypes": true,
        "forceConsistentCasingInFileNames": true,
        "isolatedModules": true,
        "jsx": "react",
        "module": "ESNext",
        "moduleResolution": "Bundler",
        "newLine": "lf",
        "noFallthroughCasesInSwitch": true,
        "noImplicitOverride": true,
        "noImplicitReturns": true,
        "noPropertyAccessFromIndexSignature": true,
        "noUncheckedIndexedAccess": true,
        "noUnusedLocals": true,
        "noUnusedParameters": true,
        "removeComments": true,
        "resolveJsonModule": true,
        "skipLibCheck": true,
        "strict": true,
        "target": "ES6",
        "verbatimModuleSyntax": true
      },
      "include": [
        "**/*.jsx"
      ]
    }

  3. Дерево элементов каталога должно приобрести следующий вид:

    .
    ├── node_modules
    │   └── ...
    ├── package-lock.json
    ├── package.json
    └── tsconfig.json

Модули и ресурсы​

  1. Создадим каталоги для модулей и ресурсов веб­‑приложения

    mkdir public
    mkdir src
    mkdir "src/components"
    echo '' > index.html
    echo '' > src/components/App.jsx
    echo '' > src/index.jsx

    Дерево элементов каталога должно приобрести следующий вид:

    .
    ├── node_modules
    │   └── ...
    ├── public
    ├── src
    │   ├── components
    │   │   └── App.jsx
    │   └── index.jsx
    ├── index.html
    ├── package-lock.json
    ├── package.json
    └── tsconfig.json

  2. Реализуем

    src/components/App.jsx
    import React from 'react';

    function App() {
      return 'Привет, мир!';
    }

    export {
      App,
    };

  3. Создадим src/index.jsx.

    src/index.jsx
    import React, { StrictMode } from 'react';
    import { createRoot } from 'react-dom/client';
    import { App } from './components/App.jsx';

    window.addEventListener('load', () => {
      // Находим в DOM элемент с атрибутом id, равным my-app-root.
      const element = document.getElementById('my-app-root');

      if (!element) {
        throw new Error('Элемент не найден.');
      }

      // Определяем его как корневой элемент для отображения компонентов React.
      const root = createRoot(element);

      // Отображаем компонент App.
      root.render(
        <StrictMode>
          <App />
        </StrictMode>
      );
    });

  4. Создадим файл index.html.

    index.html
    <!DOCTYPE html>
    <html lang="ru-luna1918">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport"
            content="width=device-width, initial-scale=1.0">
      <title>Интернет­‑магазин</title>
      <script type="module"
              src="src/index.jsx"></script>
    </head>
    <body>
      <h1>Интернет­‑магазин</h1>
      <div id="my-app-root">
        <noscript>Включите <i lang="en">JavaScript</i>.</noscript>
      </div>
    </body>
    </html>

    Внимание заслуживают

    • к веб­‑странице подключён главный модуль React;

    Элемент, который мы намерены

    Содержимое элемента noscript будет отображено, если JavaScript в пользовательском агенте (браузере) выключен или не реализован.

Пробный запуск​

  1. Запустим отладочный веб­‑сервер.

    npx vite

    Будет выведен URI (обычно http://localhost:5173/), по которому необходимо обратиться через браузер.
