Теоретические сведения

Цикломатическая сложность​

Цикломатическая сложность — мера измерения сложности программы, которую в 1976 г. предложил Томас Мак­‑Кейб. Высокая цикломатическая сложность указывает на сложность тестирования программы и не зависит от размера программы. Даже небольшая программа с высокой цикломатической сложностью может оказаться непригодной для тестирования [mccabe, с. 308].

Значение цикломатической сложности иногда называют числом Мак­‑Кейба. Его можно вычислить для произвольного фрагмента исходного текста, в т. ч. для подпрограммы (процедуры, функции, метода) [allen; mccabe].

Цикломатическая сложность обычно учитывается при тестировании базового пути — также разработанном Т. Мак­‑Кейбом методе проектирования контрольных примеров, предназначенном для тестирования программы методом белого ящика. Тестирование базового пути гарантирует полное тестовое покрытие линейно независимых путей выполнения, то есть по крайней мере однократное выполнение каждого выражения в исходном коде, с использованием минимально достаточного числа контрольных примеров.

к сведению

Цикломатическая сложность равна числу контрольных примеров, необходимых для однократного выполнения каждого линейно независимого пути в программе.

Цикломатическую сложность используют:

  • в интегрированной среде разработки Microsoft Visual Studio 2022 — в расчёте индекса удобства обслуживания программы [vs-code-mi];
  • в программных продуктах IBM, например Rational Test RealTime.

Граф потока управления​

Алгоритм вычисления цикломатической сложности программы требует построить и проанализировать граф потока управления (англ. control­‑flow graph, CFG), для которого характерны следующие элементы:

  • вершины — обозначают простые или составные предложения;
  • дуги — обозначают передачу управления.

Вершины делятся на предикатные и операторные.

Предикатная вершина обозначает простое логическое выражение (например, a∧ba \land ba∧b, n>0n > 0n>0, x1≠x2x_1 \neq x_2x1​=x2​), в зависимости от которого особая языковая конструкция определяет более одного пути выполнения программы. К таким конструкциям отнесены условный оператор (if...else), оператор ветвления (case...of, switch...case), оператор цикла (for, while) и т. п. Из предикатной вершины выходят две и только две дуги, соответствующих ложному и истинному решениям.

Предикатная вершина

К сведению

Оператор может содержать множество логических выражений (после if, после else if и т. п.), а логическое выражение может быть составным. Так, сложное логическое выражение (n>0)∧(n<10)(n > 0) \land (n < 10)(n>0)∧(n<10) составлено из трёх простых. Каждое простое логическое выражение должно быть обозначено отдельной предикатной вершиной.

Операторная вершина обозначает предложение, которое может содержать объявление переменной, оператор ввода­‑вывода, комментарий, вызов процедуры, возврат из процедуры (return) и другие языковые конструкции, кроме условного оператора, оператора цикла и оператора ветвления. Из операторной вершины (кроме конечной) выходит одна и только одна дуга.

Операторная вершина

Вход в программу может быть обозначен операторной вершиной, в которую не входит ни одна дуга. Выход из программы должен быть обозначен одной и только одной конечной операторной вершиной, из которой не выходит ни одна дуга. Таким образом, если программа пуста, то она обозначается либо одной (конечной), либо двумя (соединёнными между собой) операторными вершинами.

Формулы​

Цикломатическая сложность c(G)c(G)c(G) может быть вычислена по формулам:

  1. c(G)=e−v+2gc(G) = e - v + 2gc(G)=e−v+2g, где eee — число дуг (∣E∣|E|∣E∣), vvv — число вершин (∣V∣|V|∣V∣), ggg — число графов, т. е. вызванных подпрограмм. Для упрощения принимают g=1g = 1g=1;
  2. c(G)=v⋄+1c(G) = v_{\diamond} + 1c(G)=v⋄​+1, где v⋄v_{\diamond}v⋄​ — число предикатных вершин;
  3. c(G)=fc(G) = fc(G)=f, где fff — число граней (∣F∣|F|∣F∣), т. е. областей, ограниченных вершинами и дугами. Внешняя часть плоскости, на которой уложен граф, также образует грань [mccabe, с. 309, 314–⁠315];
  4. c(G)=∣⟨V1,Vn⃗⟩∣c(G) = |\langle \vec{V_{1}, V_{n}} \rangle|c(G)=∣⟨V1​,Vn​​⟩∣, где ∣⟨V1,Vn⃗⟩∣|\langle \vec{V_{1}, V_{n}} \rangle|∣⟨V1​,Vn​​⟩∣ — число путей от входа до выхода из программы.

Все формулам дают одинаковые результаты, поэтому их можно (и нужно!) использовать для проверки.
