Теоретические сведения

Как говорилось ранее, методика [проектирования] тестирования (англ. test [design] technique) — процедура, необходимая для создания или выбора модели тестирования, определения элементов тестового покрытия и разработки соответствующих контрольных примеров [1, п. 3.94; 2, п. 4.59]. Примеры методик: тестирование ветвей, разбиение [на классы] эквивалентности, анализ граничных значений [1, п. 3.94] и др.

Тестовое покрытие (англ. test coverage) — выраженная в процентах степень, в которой специфицированные элементы тестового покрытия были проверены контрольным примером или контрольными примерами [3, п. 4.53; 2, п. 4.53; 1, п. 3.89].

Элемент тестового покрытия (англ. test coverage item) — измеримый атрибут элемента тестирования (модуля или полной системы. ― Прим. сост.), на котором сфокусировано тестирование. Примеры элементов тестового покрытия: классы эквивалентности, переходы между состояниями, исполняемые операторы [1, п. 3.90] и др.

Неполное тестовое покрытие может означать, что

  • требования к программе неполны;
  • исходный код не связан с требованиями;
  • тесты неадекватны.

Стопроцентное тестовое покрытие не означает:

  • что исходный код корректен. Дейкстра говорил, что тестирование показывает наличие ошибок, а не их отсутствие;
  • что верны требования к программе (это валидация с заказчиком);
  • что протестированы все требования к программе;
  • что другая версия транслятора (компилятора, интерпретатора, конвертера) не приведёт к ошибкам;
  • что достигнуто стопроцентное покрытие объектного или байт­‑кода.

Тестирование методом чёрного ящика​

Эти методики предполагают проектирование контрольных примеров на основе спецификации (документации на программу), без доступа к исходному коду.

Разбиение [на классы] эквивалентности​

Разбиение [на классы] эквивалентности (англ. equivalence partitioning) — методика проектирования тестирования, при которой контрольные примеры разработаны таким образом, чтобы проверить классы (разделы) эквивалентности с помощью одного или более представительных элементов каждого класса (раздела) [3, п. 4.13; 2, п. 4.13; 1, п. 3.31].

Класс (раздел) эквивалентности (англ. equivalence class [1, п. 3.30], equivalence partition [3, п. 4.11; 2, п. 4.11; 1, п. 3.30]) — класс вводов или выводов, элементы которого, как ожидается, будут обработаны элементом тестирования одним и тем же образом [1, п. 3.30]. Слово «класс» использовано здесь в значении, принятом в математике, а не в теории объектно­‑ориентированного программирования.

Методика предоставляет эвристические данные, позволяющие «угадать», на чём тестировщику следует сосредоточить усилия [4], что повышает эффективность (вероятность нахождения ошибки при равных усилиях)?

Обычно результат разбиения субъективен [2, п. 4.12 (прим.)].

Советы по разбиению​

Рассмотрим пример. Функция Э́йлера φ(n)\varphi(n)φ(n), используемая в криптографическом алгоритме RSA, показывает, сколько натуральных (целых положительных) чисел от 111 до nnn включительно взаимно просты с nnn, т. е. имеют с ним наибольший общий делитель, равный 111.

Очевидно, что функция определена на множестве натуральных чисел — её аргумент nnn может принимать значения из интервала целых чисел [1;+∞)[1; +\infin)[1;+∞), но целые числа из интервала (−∞;0](-\infin; 0](−∞;0] и нецелые числа из интервала (−∞;+∞)(-\infin; +\infin)(−∞;+∞) недопустимы.

Каждое из этих допустимых или недопустимых числовых множеств может быть представлено как класс эквивалентности.

Как ожидается, программа, реализующая функцию Эйлера, одним и тем же образом будет обрабатывать аргументы, относящиеся к одному и тому же классу эквивалентности:

  • для целого nnn из интервала [1;+∞)[1; +\infin)[1;+∞) (например, 1, 2, 3 и т. д.) программа произведёт вычисление и выдаст результат;
  • для нецелого nnn из интервала (−∞;+∞)(-\infin; +\infin)(−∞;+∞) (например, 0,50,50,5) программа выведет сообщение об ошибке или, по выбору пользователя, округлить значение параметра и вычислить результат;
  • на интервале целых чисел (−∞;0](-\infin; 0](−∞;0] можно выделить два класса эквивалентности: (−∞;−1](-\infin; -1](−∞;−1] и [0;0][0; 0][0;0]. В первом случае программа может предложить вычислить функцию Эйлера для модуля nnn (∣n∣|n|∣n∣), а во втором случае — вывести сообщение, что ноль — недопустимый аргумент функции.

Анализ граничных значений​

Анализ граничных значений (англ. boundary value analysis) — методика проектирования тестирования на основе спецификации, основанная на проверке границ разделов (классов) эквивалентности [1, п. 3.12; 2, п. 5.5.6.2, D.10].

Дополняет методику разбиения [на классы] эквивалентности. Учитываются и входные, и выходные данные [5, с. 476].

Советы по разбиению​

  1. Если все элементы класса эквивалентности лежат в замкнутом интервале целых чисел [m;n][m; n][m;n], то тестовые данные должны содержать все элементы множества {m−1;m;n;n+1}\{m - 1; m; n; n + 1\}{m−1;m;n;n+1} [5, с. 477].
  2. Если все элементы класса эквивалентности лежат в замкнутом интервале действительных чисел [rmin;rmax][r_{min}; r_{max}][rmin​;rmax​], то тестовые данные должны содержать все элементы множества {rmin−Δ;rmin;rmax;rmax+Δ}\{r_{min} - \Delta; r_{min}; r_{max}; r_{max} + \Delta\}{rmin​−Δ;rmin​;rmax​;rmax​+Δ} [5, с. 476–⁠477], где Δ\DeltaΔ — шаг (напримет, 0,01 для интервала [5,00;7,00][5,00; 7,00][5,00;7,00]).
