Пример выполнения задания

Проектирование БД​

Выполним диаграмму «сущность — связь» (ERD).

Реализация БД​

  1. Запустим клиент СУБД MySQL:

    mysql --user=root --password

    Введём пароль пользователя БД.

    Должно быть выдано приглашение ко вводу команд:

    mysql>
    К сведению

    По умолчанию сервер MySQL, реализованный как демон (фоновый процесс) mysqld, занимает порт 3306, и клиент подаёт запросы на этот же порт. Это обусловлено следующими строками в конфигурационном файле:

    [mysqld]
    port=3306

    [client]
    port=3306

    Если мы сменили IP­‑адрес или номер порта, то их следует явно указывать при подаче команд, например

    mysql --user=root --password --bind-address=127.0.0.1 --port=3306

    Чтобы узнать номер порта, занятого сервером MySQL, подайте команду

    • Windows
    • Linux
    • macOS
    mysqld --help --verbose | findstr /b /c:"port "
    mysqld --help --verbose | grep '^port '
    mysqld --help --verbose | grep '^port '

  2. Создадим БД shop, подав команду

    CREATE SCHEMA shop
    DEFAULT CHARACTER SET utf8mb4;
    -- Сравнение `utf8mb4_0900_ai_ci` будет установлено автоматически.
    -- Удалить БД можно, подав запрос `DROP SCHEMA shop;` (без грависов).

    Должно быть выведено:

    Query OK, 1 row affected...

  3. Выведем перечень БД, находящихся под управлением MySQL, и убедимся, что БД shop создана.

    SHOW DATABASES;

    Должно быть выведено:

    +-----------+
    | Databases |
    +-----------+
    | ...       |
    | shop      |
    | ...       |
    +-----------+

  4. Переключим клиент на эту БД:

    USE shop;

Реализация сущности «Товары»​

  1. Создать таблицу позволит выражение CREATE TABLE. Возьмём его как заготовку:

    CREATE TABLE products (
    );

    Далее добавим между скобками строки, описывающие атрибуты таблицы.

BOOLEAN в MySQL — синоним

Фактически СУБД MySQLсоздаст столбец неотрицательного целочисленного типа данных BIGINT UNSIGNED с запретом на пустые значения (NOT NULL), а также создаст счётчик (AUTO_INCREMENT) и внесёт этот столбец в состав первичного ключа (PRIMARY KEY):

`BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY`

  1. Для хранения наименования товара нужен столбец name с небольшой максимальной длиной (в символах) и запретом на повтор значений:

    title VARCHAR(55) NOT NULL,
    Почему именно 55?

    1. Тип данных VARCHAR параметризован максимальной допустимой длиной строки, измеряемой в символах, но на уровне физической схемы БД длина строки измеряется в байтах с учётом набора символов — так называемой «кодировки». Например, набор utf8mb4 в MySQL 8 выделяет на каждый символ от 1 до 4 байтов: латинская буква занимает 1 байт, русская — 2 байта и т. д. При этом MySQL 8 хранит информацию о длине строки (в символах) внутри значения VARCHAR: если длина лежит в интервале [0;28−1][0; 2^{8} - 1][0;28−1], то информация занимает один старший байт; длина в интервале [28;216−1][2^{8}; 2^{16} - 1][28;216−1] означает, что будут заняты два старших байта. Строки большей длины VARCHAR не поддерживает.

      Таким образом, при использовании набора символов utf8mb4 в VARCHAR можно хранить строку, длина которой в символах не превышает

      ⌊216−14⌋=16383.\left\lfloor \frac{2^{16} - 1}{4} \right\rfloor = 16383.⌊4216−1​⌋=16383.

      При этом информация о длине строки (в символах) займёт оба старших байта. На 1 Б можно записать информацию о такой длине строки, которая в символах не превышает

      ⌊28−14⌋=63.\left\lfloor \frac{2^{8} - 1}{4} \right\rfloor = 63.⌊428−1​⌋=63.

    2. Поисковые системы ограничивают длину заголовка веб­‑страницы. Заголовок сайта МПТ в поисковой выдаче Google ограничен приблизительно 55 знаками.

      Конечно, в заголовке веб­‑страницы обычно выводят не только название самой веб­‑страницы, но и название сайта. Поэтому следует ориентироваться на число 55 (или меньшее) как на максимальную допустимую длину наименования товара.

  2. Для хранения цены товара добавим столбец price, позволяющий хранить денежные величины:

    price DECIMAL(11,2) NOT NULL,

    Такой столбец позволит хранить 11­‑разрядное число, два младших разряда которого отведены под дробную часть (например, копейки или центы). Таким образом, возможные значения столбца будут лежать в интервале [−999999999,99;999999999,99][-999999999,99; 999999999,99][−999999999,99;999999999,99]. Цены в магазине не должны превышать 1 млрд рублей минут 1 копейка.

    Справка

    Точность NUMERIC определена в стандарте SQL, а точность DECIMAL зависит от СУБД. В MySQL эти типы данных реализованы одинаково.

    Не используйте FLOAT или DOUBLE для денежных величин

    Широко известные типы данных, реализующие стандарт IEEE 754, не вполне пригодны для денежных операций, так как не обеспечивают должную точность представления чисел. Так, при использовании FLOAT или DOUBLE сумма 0,10 ₽ и 0,70 ₽ будет записана как 0,7⁠(9) ₽.

  3. Добавим столбец description для описания товара:

    description TEXT NOT NULL,

  4. Добавим столбец целого неотрицательного типа данных со значением 0 по умолчанию. В этом столбце мы будем хранить число единиц товара на складе:

    quantity INT NOT NULL DEFAULT 0,

    Если значение столбца не задано при вставке, то считается, что на складе не осталось данного товара.

  5. Так как товар может быть снят с продажи независимо от его запасов на складе, потребуется столбец бу́лева типа данных.

    available BOOLEAN NOT NULL
    BOOLEAN в MySQL — синоним

    Фактически СУБД MySQLсоздаст столбец целочисленного типа данных TINYINT. В нём значение 0 обозначает FALSE, значение 1 — TRUE.

  6. В результате получится следующее предложение:

    CREATE TABLE products (
      id SERIAL,
      name VARCHAR(55) NOT NULL,
      price DECIMAL(11, 2) NOT NULL,
      description TEXT NOT NULL,
      quantity INT UNSIGNED NOT NULL DEFAULT 0,
      available BOOLEAN
    );

    Подадим запрос на его выполнение в клиент СУБД.

  7. Подадим команду для вывода описания таблицы:

    DESCRIBE products;

    Должно быть выведено:

    +-------------+-----------------+------+-----+---------+----------------+
    | Field       | Type            | Null | Key | Default | Extra          |
    +-------------+-----------------+------+-----+---------+----------------+
    | id          | bigint unsigned | NO   | PRI | NULL    | auto_increment |
    | name        | varchar(55)     | NO   |     | NULL    |                |
    | price       | decimal(11,2)   | NO   |     | NULL    |                |
    | description | text            | NO   |     | NULL    |                |
    | quantity    | int unsigned    | NO   |     | 0       |                |
    | available   | tinyint(1)      | YES  |     | NULL    |                |
    +-------------+-----------------+------+-----+---------+----------------+
