Контейнеризация Laravel

Добавим в образ менеджер зависимостей Composer и установщик Laravel.

  1. Внесите изменения в файл php/Dockerfile.

    Файл php/Dockerfile
    FROM php:8.4-fpm-alpine

    RUN cp "${PHP_INI_DIR}/php.ini-development" "${PHP_INI_DIR}/php.ini"

    RUN apk add --no-cache -- \
            php84-pdo_mysql \
        && docker-php-ext-install \
            pdo_mysql

    COPY --from=composer:latest -- /usr/bin/composer /usr/bin/composer

    RUN composer global require laravel/installer

    Инструкция COPY копирует файл/⁠каталог в файловую систему образа. Объектом копирования могут выступать файлы из другого образа (тогда его тег указывают как значение опции --from), из каталога хостовой ОС и др. В данном случае Docker скачает образ composer:latest, скопирует файл /usr/bin/composer из его файловой системы и разместит его копию по одноимённому пути в созданном нами образе.

  2. Пересоберите образ.

    docker image remove project:php
    docker image build --tag=project:php -- ./php

  3. Создайте и запустите контейнер как утилиту, чтобы создать шаблон проекта Laravel в каталоге src.

    • macOS
    • Ubuntu
    • Windows 11
    docker container run \
      --env=COMPOSER_PROCESS_TIMEOUT=1000 \
      --rm \
      --tty \
      --volume="./src/:/var/www/html/" \
      -- \
      project:php \
      composer global exec \
        -- \
        laravel new \
          --database=mysql \
          --no-interaction \
          --phpunit \
          -- \
          laravel
    docker container run \
      --env=COMPOSER_PROCESS_TIMEOUT=1000 \
      --rm \
      --tty \
      --volume="./src/:/var/www/html/" \
      -- \
      project:php \
      composer global exec \
        -- \
        laravel new \
          --database=mysql \
          --no-interaction \
          --phpunit \
          -- \
          laravel
    docker container run ^
      --env=COMPOSER_PROCESS_TIMEOUT=1000 ^
      --rm ^
      --tty ^
      --volume="%CD%\src\:/var/www/html/" ^
      -- ^
      project:php ^
      composer global exec ^
        -- ^
        laravel new ^
          --database=mysql ^
          --no-interaction ^
          --phpunit ^
          -- ^
          laravel

    В каталоге src в хостовой ОС появится каталог laravel с файлами проекта.

    Process timeout

    Переменная окружения COMPOSER_PROCESS_TIMEOUT задаёт число секунд, которое Composer отводит на выполнение команды. По умолчанию оно равно 300, и этого времени может не хватить. Тогда выводится сообщение об ошибке: «The following exception in caused by a process timeout».

  4. Внесите изменения в файл compose.yaml.

    Файл compose.yaml
    services:
      php:
        build: ./php/
        command: php artisan serve --host=0.0.0.0 --port=8000
        container_name: project-php
        depends_on:
          - mysql
        ports:
          - "8000:8000"
        volumes:
          - ./src/laravel/:/var/www/html/laravel/:rw
        working_dir: /var/www/html/laravel/
    # Остальные строки не изменяйте.

  5. Подайте команду на сборку и запуск контейнеров:

    docker compose up --build --detach --remove-orphans

  6. Обратитесь через браузер по веб­‑адресу http://localhost:8081/, чтобы запустить phpMyAdmin.

  7. Создайте через phpMyAdmin БД laravel.

  8. Откройте на редактирование файл src/laravel/.env, найдём следующие переменные и, если требуется, приведите их значения в соответствие примеру.

    Фрагмент файла .env
    DB_CONNECTION=mysql
    DB_HOST=project-mysql
    DB_PORT=3306
    DB_DATABASE=laravel
    DB_USERNAME=root
    DB_PASSWORD=1

    Напомню, что project-mysql — это имя контейнера с MySQL. Оно выполняет функции хоста. Порт, имя пользователя и пароль указаны в настройках контейнера в файле compose.yaml.

  9. Подайте команду на запуск командной строки внутри контейнера:

    • macOS
    • Ubuntu
    • Windows 11
    docker container exec \
      --interactive \
      --tty \
      -- \
      project-php \
      /bin/sh
    docker container exec \
      --interactive \
      --tty \
      -- \
      project-php \
      /bin/sh
    docker container exec ^
      --interactive ^
      --tty ^
      -- ^
      project-php ^
      /bin/sh

    или

    docker exec -it project-php /bin/sh

  10. Находясь в гостевой ОС, подайте команду на выполнение сценариев миграции БД.

    php artisan migrate

  11. Обратитесь через браузер по веб­‑адресу http://localhost:8000/.

    Должна быть открыта главная страница Вашего веб­‑сайта с логотипом Laravel.

  12. Подайте команду на выход из гостевой ОС.

    exit

  13. Подайте команду на останов контейнеров (их состояние сохранится):

    docker compose stop

    Впоследствии их можно будет запустить командой:

    docker compose start

    При необходимости сервисы можно удалить:

    # docker compose down --remove-orphans --volumes

  14. Подайте команду на вывод перечня работающих контейнеров:

    docker compose ps

    Перечень должен быть пуст.
