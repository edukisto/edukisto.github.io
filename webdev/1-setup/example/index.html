Пример выполнения задания

Установка PHP​

  • Windows
  • Ubuntu 22.04.3
  • macOS

Установите Open Server Panel или XAMPP.

sudo LC_ALL=C.UTF-8 add-apt-repository ppa:ondrej/php

sudo apt update

# Убедимся, что пакеты PHP 8 доступны для установки.
apt search php8

# Симулируем установку интерпретатора (для проверки).
sudo apt install -s php8.3

# Установим интерпретатор.
sudo apt install php8.3

# Выведем номер установленной версии интерпретатора (для проверки).
php --version

# Установим расширение для работы с MySQL.
sudo apt install php8.3-mysql

sudo apt install php8.3-xdebug

Установите XAMPP.

Установка MySQL​

MySQL — это клиент­‑серверная система управления базами данных (СУБД). Через графический или текстовый пользовательский интерфейс, встроенный в тот или иной клиент, можно подавать команды серверу; сервер принимает запросы и присылает клиенту результаты их обработки.

  • Windows
  • Ubuntu
  • macOS

Установите Open Server Panel или XAMPP.

  1. Для установки потребуется пакет mysql-apt-config. Откройте в браузере веб­‑страницу https://dev.mysql.com/downloads/repo/apt/, где опубликованы

    • гиперссылка на пакет (кнопка «Download»);
    • эталонная контрольная сумма пакета (рядом с меткой «MD5»);
    • гиперссылка на сигнатуру (гиперссылка «Signature»).

  2. Нажмите на кнопку «Signature» и во всплывающем окне нажмите на кнопку «Download signature», чтобы скачать файл с PGP­‑сигнатурой.

  3. Нажмите на кнопку «Download», чтобы скачать пакет. Если вам предложат зарегистрироваться или авторизоваться, перейдите по гиперссылке «No thanks, just start my download».

  4. Рассчитайте контрольную сумму скачанного пакета и сверьте её с эталоном, опубликованным на сайте (рядом с меткой MD5).

    md5sum mysql-apt-config_0.8.29-1_all.deb

  5. На веб­‑странице https://dev.mysql.com/doc/refman/8.0/en/checking-gpg-signature.html опубликован идентификатор PGP­‑ключа для MySQL 8.0.36. На момент составления настоящей инструкции это A8D3785C. Импортируйте этот ключ и, используя его, сверьте подпись пакета с эталоном сигнатуры.

    # Импортируем ключ с публичного сервера.
    gpg --keyserver hkp://keyserver.ubuntu.com --recv-key A8D3785C

    # Сверяем подпись.
    gpg --verify mysql-apt-config_0.8.29-1_all.deb.asc mysql-apt-config_0.8.29-1_all.deb

    Должно быть выведено сообщение, что подпись действительна. Отсутствие доверенной подписи не помешает дальнейшей работе.

  6. Если проверки прошли успешно, установите пакет.

    sudo dpkg -i mysql-apt-config_0.8.29-1_all.deb

    Нажимая на клавиши Вверх или Вниз, установите курсор на пункт «OK» и нажмите на клавишу Enter.

    Должны быть выведены следующие сообщения:

    Распаковывается mysql-apt-config...
    Настраивается пакет mysql-apt-config...
    ...
    OK

  7. Установите СУБД.

    sudo apt update

    # Убедимся, что пакеты MySQL 8 доступны для установки.
    apt search mysql-server

    # Симулируем установку СУБД (для проверки).
    sudo apt install -s mysql-server

    # Установим СУБД.
    sudo apt install mysql-server

    # Выведем номер установленной версии СУБД (для проверки).
    mysql --version

    Установщик попросит Вас задать пароль и выбрать метод аутентификации. Выберите рекомендованный вариант.

  8. Сервер MySQL в Linux обычно реализован как демон (программа, выполняемая в фоновом режиме) mysqld, а стандартный клиент — как утилита mysql. Кроме того, существует одноимённая служба — /etc/init.d/mysql, которая управляет демоном. Запустите клиент MySQL, введя в оболочку команду

    mysql -u root -p

    и пароль. Будет выдано приглашение ко вводу команд:

    mysql>

  9. Выясните, какие пользователи СУБД внесены в системную базу данных:

    SELECT user, host, plugin, authentication_string FROM mysql.user;

    Должна быть выведена таблица, в которой указан пользователь root.

  10. Остановите клиент MySQL:

    EXIT;

    Будет выведено:

    Bye

Установите XAMPP.

Установка Composer​

Установка​

  • Windows (OSPanel)
  • Windows (XAMPP)
  • Ubuntu
  • macOS

Composer включён в дистрибутив Open Server Panel.

  1. Скачайте Composer-Setup.exe.

  2. Запустите Composer-Setup.exe от имени администратора.

  3. Флажок Developer mode в окне Installation options устанавливать не нужно. Нажмите на кнопку Next.

  4. В окне Settings check

    • нажмите на кнопку «Browse...» и укажите путь к интерпретатору PHP — php.exe;
    • установите флажок «Add this PHP to your path?»;
    • нажмите на кнопку Next.

  5. Если появится окно PHP configuration error, установите флажок Create a php.ini file и нажмите на кнопку Next.

  6. Окно Proxy settings пропустите, нажав на кнопку Next.

  7. В окне Ready to install нажмите на кнопку Install. Необходимо подключение к Интернету.

  8. В окне Information Вам предложат перезагрузить компьютер. Нажмите на кнопку Next.

  9. В окне Completing Composer setup нажмите на кнопку Finish.

  10. Перезагрузите компьютер.

  1. Скачайте последнюю стабильную версию composer.phar (latest stable) с веб­‑страницы https://getcomposer.org/download/ или выполните:

    wget https://getcomposer.org/composer-stable.phar
    # или
    curl -O https://getcomposer.org/composer-stable.phar

  2. Переместите файл, чтобы его можно было вызывать из консоли по имени.

    mkdir -p ~/.local/bin/

    . ~/.profile

    mv composer-stable.phar ~/.local/bin/composer

  3. Установите право выполнения файла как программы.

    chmod +x ~/.local/bin/composer

  4. Проверка.

    composer -V

Composer включён в дистрибутив XAMPP.

Обновление​

Запустите оболочку командной строки и подайте следующие команды.

Особенности дистрибутивов

При использовании Open Server Panel запустите его, нажмите правой кнопкой мыши на флажок в области уведомлений Windows и в контекстном меню выберите пункт «Дополнительно ⏵ Консоль».

При использовании XAMPP запустите его и нажмите на кнопку Shell на его контрольной панели.

# Запустить самообновление Composer’а.
composer self-update

# Убедиться, что установлен Composer самой свежей версии.
composer -V

Установка инсталлятора Laravel​

  1. Выполним в эмуляторе терминала:

    composer global require laravel/installer

    Просматривая журнал установки, убедитесь, что Composer использует актуальную версию PHP. Если нет, удалите установленный пакет (composer global remove laravel/installer), перенастройте систему и повторите установку.

  2. Добавим инсталлятор Laravel в PATH

    • Windows (XAMPP)
    • Ubuntu

    После успешной установки инсталлятор Laravel будет размещён в каталоге %APPDATA%\Composer\vendor\bin. Убедитесь, что указанный каталог внесён в PATH, выполнив в консоли:

    cd %APPDATA%\Composer\vendor\bin
    echo 'PATH="$HOME/.config/composer/vendor/bin:$PATH"' >> ~/.profile

    . ~/.profile

  3. Переход в каталог, где будет размещён каталог проекта

    • Windows (OSPanel)
    • Windows (XAMPP)
    • Ubuntu
    • macOS
    cd domains/localhost
    cd htdocs
    cd ~
    cd ~

  4. Создадим проект на базе Laravel

    # Убедимся, что установлен инсталлятор самой свежей версии.
    laravel -V

    # Выведем перечень команд инсталлятора Laravel.
    laravel list

    # Нас интересует команда new. Выводим справочную информацию о ней.
    laravel new --help

    # Разворачиваем проект под названием project на базе каркаса веб­‑приложений Laravel.
    laravel new project

  5. Проверим проект.

    # Спускаемся в каталог project, автоматически созданный Composer’ом.
    cd project

    # Выводим номер установленной версии Laravel.
    php artisan -V

  6. Запустим проект на встроенном в PHP отладочном веб­‑сервере.

    php artisan serve

    В окно терминала будет выведено сообщение с IP­‑адресом и портом, которые занял веб­‑сервер. Обычно это http://127.0.0.1:8000.

  7. Запустите браузер и обратитесь по указанному веб­‑адресу.

    В окне браузера Вы должны увидеть заставку Laravel.

VS Code

Если Вы не пользуетесь Open Server Panel, то в дальнейшем можно будет использовать оболочку командной строки, встроенную в VS Code.

VS Code​

Установка​

  1. Установите текстовый редактор Visual Studio Code (далее — VS Code).

    • Windows
    • Ubuntu
    • macOS

    Используйте дистрибутив с веб­‑сайта https://code.visualstudio.com/.

    snap install --classic -- code

    Используйте дистрибутив с веб­‑сайта https://code.visualstudio.com/.

  2. Запустите VS Code.

Создание проекта​

  1. Файл ⏵ Открыть папку... (File ⏵ Open folder...).

  2. Через модальное окно откройте каталог project.

  3. Файл ⏵ Сохранить рабочую область как... (File ⏵ Save workspace as...) и сохраните файл project.code-workspace в каталог project.

  4. Откройте файл project.code-workspace и убедитесь, что он выглядит следующим образом:

    {
      "folders": [
        {
          "path": "."
        }
      ],
      "settings": {
      }
    }

  5. Файл ⏵ Настройки ⏵ Параметры (File ⏵ Preferences ⏵ Settings) (⎈ Ctrl + , или ⌘ Command в macOS). Переключитесь на вкладку «Рабочая область» (Workspace). Используя строку поиска, найдите и установите галочку Mouse wheel zoom.

  6. Убедитесь, что запись добавлена в файл project.code-workspace:

    {
      "folders": [
        {
          "path": "."
        }
      ],
      "settings": {
        "editor.mouseWheelZoom": true
      }
    }

Установка расширений​

  1. Файл ⏵ Настройки ⏵ Расширения (File ⏵ Preferences ⏵ Extensions) (⎈ Ctrl + ⇧ Shift + X или ⇧ Shift + ⌘ Command + X в macOS). На основной боковой панели будет открыта вкладка «Расширения» (Extensions).

  2. В поле «Поиск расширений в Marketplace» (Search extensions in Marketplace) введите: «xdebug» (без кавычек).

  3. В поисковой выдаче найдите расширение PHP Debug от издателя Xdebug и нажмите на кнопку «Установить» (Install), расположенную рядом с ним.

  4. Добавим расширение в перечень рекомендованных для участников проекта. На настроечной странице расширения и нажмите на кнопку «⚙» (Manage), выберите в контекстном меню пункт «Добавить в рекомендации рабочей области» (Add to workspace recommendations), отметьте галочкой только пункт «Рабочая область» (Workspace) и нажмите на кнопку OK.

  5. Убедитесь, что запись добавлена в файл project.code-workspace:

    {
      "folders": [
        {
          "path": "."
        }
      ],
      "settings": {
        "editor.mouseWheelZoom": true
      },
      "extensions": {
        "recommendations": [
          "xdebug.php-debug"
        ]
      }
    }

  6. Повторите для расширение PHP Intelephense от издателя Ben Mewburn.

  7. В поле «Поиск расширений в Marketplace» (Search extensions in Marketplace) введите: @builtin php. В поисковой выдаче найдите встроенное расширение «Функции языка PHP» (PHP language features), вызовите его контекстное меню и выберите пункт «Отключить» (Disable).
