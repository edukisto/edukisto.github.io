Пример выполнения задания

git clone ссылка_на_репозиторий

Будет создан каталог, названный по имени репозитория, с каталогом .git внутри.

Исходный код программы​

  1. Создайте файл README.md.

    Файл README.md
    # Моя программа

    Программа считывает с клавиатуры число
    и выводит -1, 1 или 0 в зависимости от того,
    отрицательное это число, положительное или равно нулю.

    © Иванов И. И., 2023

  2. Создайте файл src/pos_or_neg.py.

    Файл src/pos_or_neg.py
    def pos_or_neg(value: int) -> int:
        """Отрицательное, положительное или ноль?"""
        if value < 0:
            return -1
        if value == 0:
            return 0
        return 1

  3. Создайте файл src/main.py.

    Файл src/main.py
    from pos_or_neg import pos_or_neg

    # Ввести число с клавиатуры.
    number = int(input('Введите число: '))

    # Вывести -1, 1 или 0 в зависимости от того,
    # отрицательное это число, положительное или равно нулю.
    print(pos_or_neg(number))

  4. Подайте команду на выполнение программы.

    python src/main.py

    В каталоге src появится каталог __pycache__, а внутри него — файл с расширением имени .pyc (это объектный модуль, созданный при компиляции исходного модуля pos_or_neg.py). Обычно нет смысла хранить __pycache__ и его содержимое в репозитории.

Авторство​

Перед началом работы необходимо вписать информацию о себе в локальную копию репозитория.

git config user.name ваше_имя_пользователя
git config user.email ваш_адрес_электронной_почты

Адрес электронной почты можно узнать, обратившись в личном кабинете на GitHub в меню «Settings ⏵ Emails ⏵ Keep my email addresses private».

Убедитесь, что данные записаны в файл .git/config.

Состояния​

Проверим состояние рабочего каталога. Флаг -u (--untracked-files) позволяет вывести перечень файлов, изменения в которых Git не отслеживает.

git status -u

Будет выведено:

Неотслеживаемые файлы:
(используйте «git add <файл>...», чтобы добавить в то, что будет включено в коммит)
        .gitignore
        README.md
        src/main.py
        src/pos_or_neg.py

Файл .gitignore​

Вручную фильтровать файлы, добавляемые в индекс или удаляемые из него, довольно трудоёмко. В файле .gitignore можно определить файлы, которые Git должен игнорировать.

Внимание!

Файл .gitignore не затрагивает уже отслеживаемые файлы. Необходимо сначала подать команду git rm --cached, чтобы прекратить отслеживание ненужного файла, а уж затем добавить файл в .gitignore.

Несколько слов о синтаксисе:

  • каждая непустая и незакомментированная строка в содержит шаблон пути к каталогу или файлу. Для разделения элементов пути в файловой системе используют косую черту («/»). Обратная косая черта («\») позволяет экранировать пробелы в конце строки и служебные символы («!», «#»);
  • строки, которые начинаются со знака «решётка» («#»), считаются комментариями;
  • шаблон может содержать символы ? (один любой символ, кроме «/»), * (последовательность любых символов, кроме «/»), ** (последовательность любых символов);
  • восклицательный знак в начале строки инвертирует шаблон (отменяет игнорирование).

  1. Рядом с каталогом .git создайте файл .gitignore.

  2. Попробуем использовать чёрный список игнорируемых файлов.

    Файл .gitignore
    # Игнорировать каталоги __pycache__ и их содержимое.
    __pycache__/

    # Игнорировать файлы с расширениями имён .pyc, .pyd, .pyo.
    *.py[cdo]

  3. Подайте команду

    git status --ignored -u

    Будет выведен перечень игнорируемых файлов, присутствующих в рабочем каталоге.

    Игнорируемые файлы:
    (используйте «git add -f <файл>...», чтобы добавить в то, что будет включено в коммит)
            src/__pycache__/pos_or_neg.cpython-310.pyc

  4. Теперь попробуйем белый список.

    Файл .gitignore
    # Игнорировать все файлы...
    *

    # ...кроме следующих файлов в любом каталоге,
    !.editorconfig
    !.gitattributes
    !.gitignore

    # ...кроме файлов Markdown в корневом каталоге,
    !/*.md

    # ...кроме файлов с расширением имени .py в каталоге src.
    !src/
    !src/**/
    !src/**/*.py

  5. Снова подайте команду

    git status --ignored -u

    Будет выведен перечень игнорируемых файлов, присутствующих в рабочем каталоге. Он должен соответствовать предыдущему выводу.

    Игнорируемые файлы:
    (используйте «git add -f <файл>...», чтобы добавить в то, что будет включено в коммит)
            src/__pycache__/pos_or_neg.cpython-310.pyc

  6. Фиксация изменений в локальном репозитории и отправка их на GitHub.

    git add --all
    git status
    git commit -m 'Комментарий к коммиту'
    git push
