Тестирование

Настройка​

  1. В файле phpunit.xml заменим строки

    <env name="DB_CONNECTION" value="sqlite"/>
    <env name="DB_DATABASE" value=":memory:"/>

строками

<env name="DB_CONNECTION" value="mysql" />
<env name="DB_DATABASE" value="laravel" />

Тестирование API​

  1. Подадим команду на создание тестового комплекта для контроллера ProductController.

    php artisan make:test --phpunit -- ProductControllerTest
    Внимание!

    Имя создаваемого класса должно заканчиваться словом Test, иначе инструмент тестирования не учтёт этот сценарий.

    Должен быть создан файл src/laravel/tests/Feature/ProductControllerTest.php.

  2. Откроем вышеназванный файл на редактирование.

    Файл tests/Feature/ProductControllerTest.php
    <?php

    namespace Tests\Feature;

    use Illuminate\Foundation\Testing\RefreshDatabase;
    use Illuminate\Foundation\Testing\WithFaker;
    use Tests\TestCase;

    class ProductControllerTest extends TestCase
    {
        /**
         * A basic feature test example.
         */
        public function test_example(): void
        {
            $response = $this->get('/');

            $response->assertStatus(200);
        }
    }

  3. Если нам не хочется вручную удалять из БД записи, внесённые туда при тестировании, то мы можем подключим к классу трейт, очищающий БД (именно БД, а не отдельную таблицу!) после тестирования.

    use RefreshDatabase;

  4. Удалим метод test_example.

  5. Выведем таблицу маршрутизации:

    php artisan route:list

  6. Приведём файл к следующему виду:

    Файл tests/Feature/ProductTest.php
    <?php

    namespace Tests\Feature;

    use App\Models\Product;
    use Illuminate\Foundation\Testing\RefreshDatabase;
    use Tests\TestCase;

    class ProductControllerTest extends TestCase
    {
        /**
         * Отменять изменения в БД после выполнения каждого контрольного примера.
         */
        use RefreshDatabase;

        /**
         * Проверяет статус ответа HTTP при выводе перечня товаров.
         */
        public function testIndexStatus(): void
        {
            $response = $this->get(route('products.index'));

            // Проверяет, установлен ли статус ответа 200.
            $response->assertOk();
        }

        /**
         * Проверяет тело ответа HTTP при выводе перечня товаров.
         */
        public function testIndexBody(): void
        {
            $response = $this->get(route('products.index'));

            // Проверяет, содержит ли тело ответа пустой массив JSON.
            $response->assertJson([]);
        }

        /**
         * Проверяет статус ответа HTTP при вставке записи о товаре в таблицу БД.
         */
        public function testStoreStatus(): void
        {
            // Метод factory() возвращает фабрику объектов класса Product.
            // Метод make() создаёт в ОЗУ объект класса Product.
            // Ссылка на объект записывается в переменную $product.
            $product = Product::factory()->make();

            $response = $this->post(route('products.store'), $product->toArray());

            // Проверяет, установлен ли статус ответа 201.
            $response->assertCreated();
        }

        /**
         * Проверяет факт вставки записи о товаре в таблицу БД.
         */
        public function testStore(): void
        {
            $product = Product::factory()->make();
            $data = $product->toArray();
            $this->post(route('products.store'), $data);
            $this->assertDatabaseHas($product->getTable(), $data);
        }

        /**
         * Проверяет тело ответа HTTP при выводе записи о товар.
         */
        public function testShowBody(): void
        {
            $product = Product::factory()->make();
            $data = $product->toArray();
            $product->save();
            $response = $this->get(route('products.show', [
                'product' => $product->id,
            ]));
            $response->assertJson($data, false);
        }

        /**
         * Проверяет факт изменения записи о товаре в таблице БД.
         */
        public function testUpdate(): void
        {
            // Создаём объект в ОЗУ и кортеж в БД.
            $oldProduct = Product::factory()->create();

            // Создаём ещё один объект, но только в ОЗУ.
            $newProduct = Product::factory()->make();

            // Обновляем кортеж данными из второго объекта.
            $this->patch(
                route('products.update', [
                    'product' => $oldProduct->id,
                ]),
                $newProduct->toArray()
            );

            // Проверяем, обновлён ли кортеж.
            $this->assertDatabaseHas(
                $oldProduct->getTable(),
                $newProduct->toArray(),
            );
        }

        /**
         * Проверяет статус ответа HTTP при изменении записи о товаре в таблице БД.
         */
        public function testUpdateStatus(): void
        {
            $oldProduct = Product::factory()->create();
            $newProduct = Product::factory()->make();
            $response = $this->patch(
                route('products.update', [
                    'product' => $oldProduct->id,
                ]),
                $newProduct->toArray()
            );
            $response->assertStatus(204);
        }

        // public function testDestroy(): void {}

        // public function testDestroyStatus(): void {}
    }
    Внимание!

    Все методы, реализующие контрольные примеры, должны начинаться со слова test, иначе инструмент тестирования не найдёт их.

  7. Выполним тестирование, запустив ⁠PHPUnit:

    • Windows
    • Linux
    • macOS
    vendor\bin\phpunit
    ./vendor/bin/phpunit
    ./vendor/bin/phpunit

    Должно быть выведено

    OK...
Дополнительное задание

Если Вы претендуете на оценку «отлично», самостоятельно доработайте тестовый комплект, добавив в него методы для автоматизированного тестирования операции API, реализованной методом destroy.

Тестирование моделей​

  1. Подадим команду на создание тестового комплекта для модели Product.

    php artisan make:test --phpunit -- ProductTest

  2. Приведём файл к следующему виду:

    Файл tests/Feature/ProductTest.php
    <?php

    namespace Tests\Feature;

    use App\Models\Product;
    use Illuminate\Foundation\Testing\RefreshDatabase;
    use Tests\TestCase;

    class ProductTest extends TestCase
    {
        use RefreshDatabase;

        /**
         * Тест вставки кортежа в таблицу БД.
         */
        public function testInsert(): void
        {
            // Метод factory() возвращает фабрику объектов класса Product.
            // Метод create() создаёт соответствующий кортеж в БД.
            // Ссылка на объект записывается в переменную $manufacturer.
            $product = Product::factory()->create();

            // assertDatabaseHas() проверяет таблицу на наличие указанных данных.
            // $product->getTable() возвращает имя таблицы БД — products.
            // $product->toArray() возвращает массив значений свойств.
            $this->assertDatabaseHas($product->getTable(), $product->toArray());
        }

        /**
         * Тестирование изменения записи о товаре.
         */
        public function testUpdate(): void
        {
            // Создаём объект в ОЗУ и кортеж в БД.
            $oldProduct = Product::factory()->create();

            // Создаём ещё один объект, но только в ОЗУ.
            $newProduct = Product::factory()->make();

            // Обновляем кортеж данными из второго объекта.
            $oldProduct->update($newProduct->toArray());

            // Проверяем, обновлён ли кортеж.
            $this->assertDatabaseHas(
                $oldProduct->getTable(),
                $newProduct->toArray(),
            );
        }

        /**
         * Тестирование удаления записи о товаре.
         */
        public function testDestroy(): void
        {
            // Создаём объект в ОЗУ и кортеж в БД.
            $product = Product::factory()->create();

            // Удаляем кортеж.
            $product->delete();

            // Проверяем, удалён ли кортеж.
            $this->assertDatabaseMissing($product->getTable(), $product->toArray());
        }
    }

  3. Выполним тестирование, запустив ⁠PHPUnit.

    Должно быть выведено

    OK...
