Пример выполнения

Настройка сервера​

  1. Создадим корневой каталог веб­‑сайта.

    • Windows 7 SP 1, 11
    • Ubuntu
    1. Запустим контрольную панель XAMPP.
    2. Нажмём на кнопку Explorer. Будет открыта папка, в которую установлен XAMPP.
    3. Откроем папку htdocs и создадим внутри неё папку shop.example.

    Создадим в каталог shop.example.

    sudo mkdir /var/www/shop.example

  2. Создадим виртуальный хост.

    • Windows 7 SP 1, 11
    • Ubuntu

    Найдём в папке, в которую установлен XAMPP, файл apache\conf\extra\httpd-vhosts.conf и добавим в него следующую конфигурацию.

    Файл apache\conf\extra\httpd-vhosts.conf в папке XAMPP
    <VirtualHost 127.0.1.1:80>
      ServerName shop.example
      DocumentRoot "C:\xampp\htdocs\shop.example"
    </VirtualHost>

    1. Создадим файл shop.example.conf.

      Файл shop.example.conf
      <VirtualHost 127.0.1.1:80>
        ServerName shop.example
        DocumentRoot "/var/www/shop.example/"
      </VirtualHost>

    2. Разместим файл shop.example.conf в каталоге /etc/apache2/sites-available/.

    sudo cp shop.example.conf /etc/apache2/sites-available/
    «\» или «/»?

    Символ, разделяющий имена элементов файловой системы в записи пути, зависит от операционной системы (ОС):

    • в ОС, ориентируемых на семейство стандартов POSIX, в т. ч. Linux и macOS, используют косую черту («/»; англ. slash);

    • в Windows основным разделителем является обратная косая черта («\»; англ. backslash), а дополнительным (альтернативным) — обычная косая черта.

    Путь к каталогу (папке) принято завершать этим же символом, например: C:\Windows\, /etc/ и т. п. Однако обратная косая черта в конце любой строки, обозначающей путь в конфигурационном файле Apache, является синтаксической ошибкой. Это правило касается в частности директивы DocumentRoot:

    AH00526: Syntax error... DocumentRoot must be a directory

    Исправить ошибку можно двумя способами:

    • удалить обратную косую черту («\»), размещённую в конце строки;
    • заменить все обратные косые черты́ («\») в строке на обычные косые черты́ («/»).

  3. Перезапустим Apache после правки конфигурации.

    • Windows 7 SP 1, 11
    • Ubuntu
    1. Переключимся на контрольную панель XAMPP.
    2. Если напротив метки Apache видна кнопка Stop, то нажмём на неё и дождёмся останова сервера.
    3. Нажмём на кнопку Start, размещённую напротив метки Apache.
    # Активируем сайты.
    sudo a2ensite shop.example.conf

    # Заставляем Apache заново считать конфигурацию.
    sudo systemctl reload apache2

  4. Откроем на редактирование файл, соотносящий IP­‑адреса с доменными именами.

    • Windows 7 SP 1, 11
    • Ubuntu

    1. Запустим приложение «Командная строка» (не «Терминал»!) от имени администратора.

    2. Подадим команду

      Notepad "%SystemRoot%\System32\drivers\etc\hosts"

      или, если у нас установлен VS Code,

      code "%SystemRoot%\System32\drivers\etc\hosts"

      Соответствующий шаблону файл (обычно это C:\Windows\System32\drivers\etc\hosts) будет открыт в текстовом редакторе «Блокнот».

    1. Запустим эмулятор терминала.

    2. Подадим команду

      sudo nano /etc/hosts

      Файл /etc/hosts будет открыт в текстовом редакторе nano, запущенном от имени суперпользователя.

  5. Добавим в конец файла:

    127.0.1.1 shop.example

  6. Сохраним изменения и закроем текстовый редактор.

  7. Подадим команду:

    ping shop.example

    При обмене пакетами с указанным узлом вычислительной сети должен быть выведен IP­‑адрес 127.0.1.1, а доля потерянных пакетов должна составлять 0%.

    ping вместо nslookup

    Утилита nslookup в Windows (в отличие от Linux) обращается к DNS, не учитывая записи из файла hosts, поэтому она не подойдёт для наших экспериментов.

  8. Откроем в браузере вкладку с http://shop.example/.

    utils/connect_to_database.php
    <?php

    function connectToDatabase()
    {
        $databaseName = 'shop';
        $host = 'localhost';
        $password = '';
        $port = 3306;
        $username = 'root';
        $dsn = "mysql:host={$host};port={$port};dbname={$databaseName}";
        return new PDO($dsn, $username, $password);
    }
    products/index.php
    <?php

    require_once '../utils/connect_to_database.php';

    $connection = connectToDatabase();

    $sql = 'SELECT * FROM products';

    // Подаём запрос, сохраняем результат.
    $statement = $connection->query($sql);

    // Преобразуем результат в массив объектов.
    $products = $statement->fetchAll(PDO::FETCH_OBJ);

    require_once 'index.template.php';

Внешние ключи​

Выпадающий список в HTML представлен элементом­‑родителем select и множеством его дочерних элементов option. Родителю назначают атрибут name, опциям — атрибуты value. При предъявлении формы возникнет параметр с именем, равным значению атрибута name, и значением, взятым из атрибута value выбранной опции.

Рассмотрим пример реализации выпадающего списка.

<select name="category_id">
  <option value="1">Кондитерские изделия</option>
  <option value="2">Молочные продукты</option>
  <option value="3">Мясные продукты</option>
</select>

Здесь атрибуту name задано значение category_id — это имя атрибута со внешним ключом в подчинённой таблице (products). Значениями value выступают значения первичных ключей из главной таблицы (categories). Между тегами <option>...</option> размещены наименования товарных категорий. При предъявлении такой формы методом POST на сервере будет создана переменная $_POST['category_id'] со значением 1, 2 или 3 (в зависимости от выбора пользователя).

Впрочем, статический список категорий нас вряд ли устроит, если есть возможность заполнять список динамически, извлекая записи о категориях из таблицы categories.

  1. Предполагается, что главная таблица у нас уже есть:

    CREATE TABLE `categories` (
      id SERIAL,
      name VARCHAR(255) NOT NULL
    );

    Подчинённая таблица выглядит так:

    CREATE TABLE products (
      id SERIAL,
      category_id BIGINT UNSIGNED NOT NULL,
      title VARCHAR(255) NOT NULL,
      price DECIMAL(11,2) NOT NULL
    );

  2. Веб­‑страница, на которой размещена форма для создания записи о товаре (см. файл products/new.html; возможно, на занятиях мы назвали его products/create.html), сейчас сугубо статическая — её разметка хранится в файле с расширением имени .html. Мы не сможем подать запрос к СУБД и изменить содержимое документа HTML без языка программирования. Следовательно, нам придётся заменить products/new.html (или products/create.html) двумя файлами:

    • products/new.php — в нём сделаем выборку из таблицы categories;
    • products/new.template.php — в нём мы создадим форму, на которой разместим выпадающий список.

  3. Сразу поменяйте ссылку на создание записи в файле index.template.php.

    Было:

    <a href="/products/new.html">Создать запись о товаре</a>

    Станет:

    <a href="/products/new.php">Создать запись о товаре</a>

  4. Содержимое файла products/new.php — полностью новое.

    products/new.php
    <?php

    require_once '../utils/connect_to_database.php';

    $connection = connectToDatabase();

    $sql = 'SELECT * FROM categories';

    // Подаём запрос, сохраняем результат.
    $statement = $connection->query($sql);

    // Преобразуем результат в массив объектов.
    $categories = $statement->fetchAll(PDO::FETCH_OBJ);

    require_once 'new.template.php';

  5. Всё, что было в файле products/new.html (products/create.html), перенесём в файл products/new.template.php и дополним списком категорий. Мы уже использовали подобный приём, выводя список продуктов (см. файлы products/index.php и products/index.template.php).

            <option value="<?= $category->id ?>">
              <?= $category->name ?>
            </option>
            <?php } ?>
          </select>

    $connection = connectToDatabase();

    $sql = "INSERT INTO products(
                category_id,
                title,
                price
            ) VALUES (
                {$_POST['category_id']},
                '{$_POST['title']}',
                {$_POST['price']}
            )";

    // Выражение {$_POST['title']} в кавычках, потому что строкового типа.

    $connection->query($sql);

    header('Location: /products');
